🔧 PowerShell Level 3: Hardening & System Administration

Secure Windows systems like a pro. Learn to manage users, configure firewalls, harden services, and implement security baselines — all from PowerShell.

💡 Prerequisite: PowerShell Levels 1 & 2 (basics + incident response). These cmdlets change system configuration — test in a VM first!

👤 Local User Management (Who Has Access?)

CmdletSyntaxDescriptionExample & Explanation
Get-LocalUser Get-LocalUser [[-Name] username] List all local user accounts Get-LocalUser | Where-Object {$_.Enabled -eq $true}
→ Shows only enabled (active) user accounts
Get-LocalUser -Name "Administrator" | Select-Object Name, Enabled, PasswordLastSet
→ Check if default Administrator is still enabled (security risk!)
New-LocalUser New-LocalUser -Name username -Password (Read-Host -AsSecureString) [-PasswordNeverExpires] Create a new local user account $password = Read-Host -AsSecureString
New-LocalUser -Name "secuser" -Password $password -FullName "Security User" -Description "For security monitoring"
Translation: Creates a new user with a secure password prompt.
Add-LocalGroupMember Add-LocalGroupMember -Group "GroupName" -Member "username" Add a user to a local group (e.g., Administrators) Add-LocalGroupMember -Group "Administrators" -Member "secuser"
Translation: "Grant secuser administrator privileges."
⚠️ Warning: Only add trusted users to Administrators group!
🔐 User Management Security Best Practices:
• Get-LocalUser | Where-Object {$_.PasswordNeverExpires -eq $true} → Find accounts with never-expiring passwords
• Disable-LocalUser -Name "Guest" → Disable built-in Guest account
• Rename-LocalUser -Name "Administrator" -NewName "admin_secure_2024" → Rename default admin

👥 Local Group Management (Who Has What Privileges?)

CmdletSyntaxDescriptionExample & Explanation
Get-LocalGroup Get-LocalGroup [[-Name] groupname] List all local security groups Get-LocalGroup | Select-Object Name, Description
Get-LocalGroupMember Get-LocalGroupMember -Group "GroupName" List members of a specific local group Get-LocalGroupMember -Group "Administrators"
→ Shows all users with admin privileges — audit this regularly!

🔥 Windows Defender Firewall (Block the Bad Guys)

CmdletSyntaxDescriptionExample & Explanation
Get-NetFirewallRule Get-NetFirewallRule [-DisplayName name] [-Direction Inbound\|Outbound] List all firewall rules Get-NetFirewallRule -Direction Inbound | Where-Object {$_.Enabled -eq $true} | Select-Object DisplayName, Action
→ Shows all active inbound rules (what's allowed into your machine!)
New-NetFirewallRule New-NetFirewallRule -DisplayName "Name" -Direction Inbound -Protocol TCP -LocalPort port -Action Allow\|Block Create a new firewall rule New-NetFirewallRule -DisplayName "Allow SSH" -Direction Inbound -Protocol TCP -LocalPort 22 -Action Allow
→ Opens port 22 for SSH (Secure Shell)
New-NetFirewallRule -DisplayName "Block Malicious IP" -Direction Inbound -RemoteAddress 203.0.113.5 -Action Block
→ Blocks a specific malicious IP address
Set-NetFirewallProfile Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled True Enable/disable firewall profiles Set-NetFirewallProfile -All -Enabled True
→ Enables firewall on all profiles (Domain, Private, Public)
Set-NetFirewallProfile -Profile Public -DefaultInboundAction Block
→ Blocks all inbound traffic on public networks (coffee shop security!)
⚠️ Warning: If you create a "Block all" rule without allowing RDP/SSH first, you can lock yourself out of a remote server! Always test rules locally first.

⚙️ Service Management (What's Running in the Background?)

CmdletSyntaxDescriptionExample & Explanation
Get-Service Get-Service [[-Name] servicename] [-Status Running\|Stopped] List Windows services and their status Get-Service | Where-Object {$_.Status -eq "Running"} | Select-Object Name, DisplayName
→ Shows all running services (attack surface!)
Stop-Service Stop-Service -Name "servicename" [-Force] Stop a running service Stop-Service -Name "Spooler"
→ Stops the print spooler service
Set-Service -Name "Fax" -StartupType Disabled
→ Disables unnecessary services (reduces attack surface)
🔒 Services to disable on a secure system:
• Remote Registry (remote registry access — security risk)
• Print Spooler (if no printers — PrintNightmare vulnerability)
• Fax (obsolete)
• Xbox Live services (not needed on servers)

📋 System Hardening Checklist (Copy-Paste Ready)

✅ Run these commands on a new Windows system:

1. List all users and check for suspicious accounts:
Get-LocalUser | Select-Object Name, Enabled, PasswordLastSet, PasswordNeverExpires

2. Audit local administrators group:
Get-LocalGroupMember -Group "Administrators"

3. Remove unnecessary users from Administrators group:
Remove-LocalGroupMember -Group "Administrators" -Member "olduser"

4. Enable firewall on all profiles:
Set-NetFirewallProfile -All -Enabled True

5. Block all inbound traffic by default on Public networks:
Set-NetFirewallProfile -Profile Public -DefaultInboundAction Block

6. Disable unnecessary services:
Set-Service -Name "RemoteRegistry" -StartupType Disabled
Stop-Service -Name "RemoteRegistry" -Force

7. Enable PowerShell logging for security auditing:
Set-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\Windows\PowerShell\ScriptBlockLogging" -Name "EnableScriptBlockLogging" -Value 1 -Type DWord

📝 Practice Exercises

Exercise 1: Create a new local user named "auditor" with a secure password, add them to the "Remote Management Users" group, and disable password expiration
Solution:
$password = Read-Host -AsSecureString
New-LocalUser -Name "auditor" -Password $password -PasswordNeverExpires
Add-LocalGroupMember -Group "Remote Management Users" -Member "auditor"
Exercise 2: Create a firewall rule that allows inbound SSH (port 22) only from the IP range 192.168.1.0/24
Solution:
New-NetFirewallRule -DisplayName "Allow SSH from LAN" -Direction Inbound -Protocol TCP -LocalPort 22 -RemoteAddress 192.168.1.0/24 -Action Allow
Exercise 3: Find all services that are currently running with their startup type set to "Automatic"
Solution:
Get-Service | Where-Object {$_.Status -eq "Running" -and $_.StartType -eq "Automatic"} | Select-Object Name, DisplayName

🎓 Key Takeaways (PowerShell Hardening & Admin)

  • ✅ Get-LocalUser → Audit local accounts, disable unused ones
  • ✅ Get-LocalGroupMember -Group "Administrators" → Know who has admin rights
  • ✅ New-NetFirewallRule → Create granular firewall rules (allow/block by IP, port, program)
  • ✅ Set-NetFirewallProfile -Enabled True → Enable firewall on all profiles (critical security step)
  • ✅ Get-Service → Find unnecessary running services and disable them
  • ✅ Disable services like RemoteRegistry, Print Spooler — reduces attack surface
  • ✅ Always test hardening changes in a VM first — avoid locking yourself out!
