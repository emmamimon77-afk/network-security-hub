📝 Linux Level 4: Text Processing & Log Analysis

Master the art of manipulating text, extracting data, and analyzing logs. These commands are essential for security analysts, DevOps engineers, and anyone working with large amounts of data.

💡 Prerequisite: Linux Levels 1-3 (basics, security, hardening). These commands are often combined using pipes (`|`) for powerful data processing.

🔍 grep — Advanced Pattern Searching

grep (Global Regular Expression Print) finds lines matching a pattern. It's the Swiss Army knife of log analysis.

CommandSyntaxDescriptionExample & Explanation
grep (basic) grep "error" logfile Find lines containing "error" grep "Failed password" /var/log/auth.log
→ Finds all failed login attempts in the authentication log.
Translation: "Show me every line that contains 'Failed password'."
grep -i grep -i "warning" logfile Case-insensitive search grep -i "error" /var/log/syslog
→ -i ignores case, so it matches "error", "Error", "ERROR", etc.
Use case: When you're not sure if the log uses uppercase or lowercase.
🔧 grep with pipes — Counting occurrences:
cat /var/log/auth.log | grep "Failed password" | wc -l
→ cat outputs the file, grep filters lines, wc -l counts them.
Result: Total number of failed login attempts.
📖 Regular Expression (Regex) Quick Reference for grep:
• ^ Start of line: grep "^2024" logfile (lines starting with "2024")
• $ End of line: grep "error$" logfile (lines ending with "error")
• . Any single character: grep "c.t" logfile (cat, cut, cot, etc.)
• * Zero or more of previous: grep "fe*d" logfile (fd, fed, feed, feeed)
• [ ] Character class: grep "[0-9]" logfile (lines with any digit)
• \| Alternation (OR): grep "error\|warning\|critical" logfile

✂️ sed — Stream Editor (Find & Replace)

sed performs text transformations on a stream of data. Perfect for editing files or command output without opening an editor.

CommandSyntaxDescriptionExample & Explanation
sed s/// sed 's/old/new/' file Substitute/replace first occurrence per line echo "hello world" | sed 's/world/universe/'
→ Output: "hello universe"
Translation: "Replace the word 'world' with 'universe'."
sed 's///g' sed 's/old/new/g' file Replace ALL occurrences (global) per line echo "one two three two" | sed 's/two/2/g'
→ Output: "one 2 three 2"
Why `g`? Without `g`, only the first "two" would be replaced.
sed -i (in-place) sed -i 's/old/new/g' file Edit file directly (no output to terminal) sed -i 's/old-password/new-password/g' config.ini
→ Directly modifies config.ini file.
⚠️ Warning: Always test with `sed 's/...'` (no `-i`) first to see what will change!
💪 Real-world sed example — Log anonymization:
sed 's/\([0-9]\{1,3\}\.\)\{3\}[0-9]\{1,3\}/REDACTED/g' access.log
→ Replace all IP addresses with "REDACTED". Perfect for sharing logs publicly!

🔢 awk — Text Processing Powerhouse

awk is a complete programming language for text processing. It splits lines into fields and performs actions on them.

CommandSyntaxDescriptionExample & Explanation
awk (print field) awk '{print $1}' file Print first field (column) of each line echo "192.168.1.1 user root" | awk '{print $1}'
→ Output: "192.168.1.1"
What's happening: awk splits the line into fields separated by whitespace. $1 = first field, $2 = second, $NF = last field.
awk (filter + print) awk '/pattern/ {print $2}' file Find lines matching pattern, then print second field grep "Failed password" /var/log/auth.log | awk '{print $9}'
→ From lines about failed passwords, extract the 9th field (the source IP address).
Use case: Extract attacker IPs from authentication logs.
🔧 Building an attacker IP list with awk and sort:
sudo grep "Failed password" /var/log/auth.log | awk '{print $9}' | sort | uniq -c | sort -nr | head -10
This pipeline does: filter → extract IP → sort → count duplicates → sort by highest count → show top 10.
Result: The most active brute-forcers targeting your server!
💪 Advanced awk — Custom delimiters and arithmetic:
awk -F',' '{sum += $3} END {print sum}' data.csv
→ -F',' sets comma as field separator, sum += $3 adds the third column, END {print sum} prints total after all lines.
Use case: Calculate total bytes transferred from a CSV log.

✂️ cut, sort, uniq, wc — Data Organizers

CommandSyntaxDescriptionExample & Explanation
cut cut -d',' -f1,3 data.csv Extract specific columns/fields echo "2024-01-15,192.168.1.1,GET /index.html" | cut -d',' -f2
→ Output: "192.168.1.1"
-d',' sets comma as delimiter, -f2 selects the second field.
sort sort -nr file Sort lines (numeric, reverse order) echo -e "5\n1\n9\n3" | sort -nr
→ Output: 9, 5, 3, 1
-n (numeric sort), -r (reverse order, highest first).
uniq uniq -c file Report or filter out repeated lines echo -e "apple\nbanana\napple" | sort | uniq -c
→ Output: "2 apple", "1 banana"
Note: Input must be sorted before `uniq`, or it won't detect duplicates correctly!
wc wc -l file Count lines, words, or characters wc -l /var/log/auth.log
→ Output: "2047 /var/log/auth.log" (total lines in log)
-l (lines), -w (words), -c (characters).

🔗 Real-World Pipeline Examples (Putting It All Together)

📊 Example 1: Top 10 IP addresses accessing your web server
cat /var/log/nginx/access.log | awk '{print $1}' | sort | uniq -c | sort -nr | head -10
→ cat outputs log, awk extracts first field (IP), sort organizes, uniq -c counts, sort -nr orders by highest count, head -10 shows top 10.
📊 Example 2: Find the most common HTTP status codes
cat /var/log/nginx/access.log | awk '{print $9}' | sort | uniq -c | sort -nr
→ $9 is typically the HTTP status code (200, 404, 500, etc.). Reveals error patterns.
📊 Example 3: Extract and analyze SSH brute-force attempts by IP
sudo grep "Failed password" /var/log/auth.log | awk '{print $(NF-3)}' | sort | uniq -c | sort -nr | head -20
→ $(NF-3) means "the 4th field from the end" (source IP address). Shows you which IPs are attacking you most.

📝 Practice Exercises

Exercise 1: Extract all email addresses from a text file
Solution using grep:
grep -E -o "\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b" file.txt
-E (extended regex), -o (only matching part, not whole line)
Exercise 2: Count how many unique IPs visited your website today
Solution:
grep "$(date '+%d/%b/%Y')" /var/log/nginx/access.log | awk '{print $1}' | sort | uniq | wc -l
→ Filters today's date, extracts IPs, sorts, removes duplicates, counts.
Exercise 3: Replace all occurrences of "http://" with "https://" in a configuration file
Solution using sed:
sed -i 's|http://|https://|g' config.conf
→ Uses | as delimiter (instead of /) to avoid escaping slashes.
Exercise 4: Find the top 5 users who made the most sudo attempts
Solution:
sudo journalctl _COMM=sudo | grep -oP 'COMMAND=\K[^ ]+' | sort | uniq -c | sort -nr | head -5
→ This extracts usernames from sudo logs and counts their commands.

🎓 Key Takeaways (Text Processing Mastery)

  • ✅ grep → Search for patterns (use `-i` for case-insensitive, `-v` for invert, `-r` for recursive)
  • ✅ sed → Find and replace (test with `sed 's/.../'` before using `sed -i`)
  • ✅ awk → Extract fields ( `$1` = first, `$NF` = last, `$(NF-1)` = second to last)
  • ✅ cut → Simple column extraction (best for CSV/delimited data)
  • ✅ sort → Organize data (use `-n` for numbers, `-r` for reverse)
  • ✅ uniq -c → Count duplicates (requires `sort` first!)
  • ✅ Pipes (`|`) → Connect commands to build powerful data processing pipelines
