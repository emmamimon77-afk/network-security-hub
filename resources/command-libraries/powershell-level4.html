🏢 PowerShell Level 4: Active Directory & Enterprise Security

Master Active Directory security auditing, user management, and threat detection. These cmdlets require the Active Directory module and appropriate domain privileges.

💡 Prerequisite: PowerShell Levels 1-3. These commands require the ActiveDirectory module and domain admin or delegated permissions.
⚠️ Important: Install RSAT (Remote Server Administration Tools) first:
Add-WindowsCapability -Name "Rsat.ActiveDirectory.DS-LDS.Tools~~~~0.0.1.0" -Online

📦 Step 1: Import the Active Directory Module

Import-Module ActiveDirectory
💡 Pro tip: Run PowerShell as Administrator and ensure RSAT is installed. On Windows 10/11, this module is available but not loaded by default.

👤 Active Directory User Management

CmdletSyntaxDescriptionExample & Explanation
Get-ADUser Get-ADUser [-Identity] username [-Filter filter] [-Properties *] Get one or more AD user accounts Get-ADUser -Identity "john.doe" -Properties *
→ Shows ALL properties of user john.doe (last logon, password last set, groups, etc.)
Get-ADUser -Filter {Enabled -eq $true} -Properties LastLogonDate | Select-Object Name, LastLogonDate
→ Shows all enabled users and when they last logged in
Search-ADAccount Search-ADAccount [-AccountInactive] [-AccountExpired] [-LockedOut] [-PasswordNeverExpires] Find accounts with specific status (critical for security audits) Search-ADAccount -AccountInactive -TimeSpan 90.00:00:00 | Select-Object Name, LastLogonDate
→ Find inactive users (not logged in for 90+ days) — security risk!
Search-ADAccount -LockedOut | Select-Object Name, LockedOut
→ Find all locked-out accounts (possible brute-force attacks)
📋 Security Audit: Find high-risk accounts:
Search-ADAccount -PasswordNeverExpires | Select-Object Name, Enabled
→ Accounts with passwords that never expire (violates security policy)
Get-ADUser -Filter {Enabled -eq $true -and PasswordNeverExpires -eq $true} | Select-Object Name, PasswordNeverExpires

👥 Active Directory Group Management (Privilege Auditing)

CmdletSyntaxDescriptionExample & Explanation
Get-ADGroup Get-ADGroup [-Identity] groupname [-Properties *] Get AD group information Get-ADGroup -Identity "Domain Admins" -Properties Members
→ Shows who is in the Domain Admins group (critical security audit!)
Get-ADGroupMember Get-ADGroupMember [-Identity] groupname [-Recursive] List all members of an AD group Get-ADGroupMember -Identity "Domain Admins" | Select-Object Name, ObjectClass
→ Shows every user and group in Domain Admins (recursive)
Get-ADGroupMember -Identity "Enterprise Admins" -Recursive
→ Finds all nested members — critical for privilege auditing
⚠️ Common AD Attack: Over-privileged accounts
Get-ADGroupMember -Identity "Domain Admins" | ForEach-Object { Get-ADUser -Identity $_.SamAccountName -Properties LastLogonDate, PasswordLastSet }
→ Audit each Domain Admin: when did they last logon? When did they last change password?

🖥️ Active Directory Computer Management

CmdletSyntaxDescriptionExample & Explanation
Get-ADComputer Get-ADComputer [-Identity] computername [-Filter filter] [-Properties *] Get domain-joined computers Get-ADComputer -Filter {Enabled -eq $true} -Properties Name, OperatingSystem, LastLogonDate
→ Shows all active computers in the domain with OS and last logon
Get-ADComputer -Identity "SRV-FILE-01" -Properties * | Select-Object Name, OperatingSystem, IPv4Address
Get-ADComputer -Filter (advanced) Get-ADComputer -Filter "OperatingSystem -like '*Server*'" Filter computers by OS, location, or other attributes Get-ADComputer -Filter "OperatingSystem -notlike '*Server*'" | Select-Object Name, OperatingSystem
→ Find all workstations (non-server OS) for desktop management

🚨 Detecting Common Active Directory Attacks

🔴 Attack 1: Kerberoasting
Attackers request service tickets (TGS) for accounts with SPNs and crack them offline.
Detection:
Get-ADUser -Filter {ServicePrincipalName -like "*"} -Properties ServicePrincipalName, Name, PasswordLastSet | Select-Object Name, ServicePrincipalName, PasswordLastSet
→ Lists all accounts with SPNs (potential Kerberoasting targets). Investigate accounts with old passwords!
🔴 Attack 2: Golden Ticket
Attackers forge Kerberos tickets after compromising KRBTGT account.
Detection/Mitigation:
Get-ADUser -Identity krbtgt -Properties PasswordLastSet, LastLogonDate
→ Check when KRBTGT password was last changed. Microsoft recommends changing twice!
Monitor Event ID 4769 for unusual TGS requests.
🔴 Attack 3: Password Spraying
Attackers try common passwords across many accounts to avoid lockout.
Detection:
Search-ADAccount -LockedOut | Select-Object Name, LockedOut
→ Few locked accounts? Good. Many locked at once? Possible password spray.
Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4625} | Where-Object {$_.Message -like "*Account locked out*"} | Group-Object -Property User | Select-Object Count, Name
🔴 Attack 4: DCShadow
Attackers register a rogue DC and push malicious changes.
Detection:
Get-ADComputer -Filter {OperatingSystem -like "*Domain Controller*"} -Properties Name, IPv4Address, Modified
→ Monitor for unexpected domain controllers (Event ID 4928, 4929)

📋 Active Directory Security Audit Checklist

✅ Weekly Security Checks:

1. Check for inactive user accounts:
Search-ADAccount -AccountInactive -TimeSpan 90.00:00:00 | Select-Object Name, LastLogonDate

2. Audit privileged group membership:
Get-ADGroupMember -Identity "Domain Admins" | Select-Object Name, ObjectClass
Get-ADGroupMember -Identity "Enterprise Admins" | Select-Object Name, ObjectClass
Get-ADGroupMember -Identity "Schema Admins" | Select-Object Name, ObjectClass

3. Find accounts with never-expiring passwords:
Search-ADAccount -PasswordNeverExpires | Select-Object Name, Enabled

4. Check for locked-out accounts (possible attacks):
Search-ADAccount -LockedOut | Select-Object Name, LockedOut

5. Verify KRBTGT password age:
Get-ADUser -Identity krbtgt -Properties PasswordLastSet
→ Should be changed every 6-12 months (twice with delay)

6. List all domain controllers:
Get-ADDomainController -Filter * | Select-Object Name, IPv4Address, Site

📝 Practice Exercises

Exercise 1: Find all users who haven't logged in for 60+ days and disable them
Solution:
$inactiveUsers = Search-ADAccount -AccountInactive -TimeSpan 60.00:00:00
$inactiveUsers | Disable-ADAccount
$inactiveUsers | Select-Object Name, LastLogonDate
Exercise 2: Export all members of the "Domain Admins" group to a CSV file for auditing
Solution:
Get-ADGroupMember -Identity "Domain Admins" | Select-Object Name, ObjectClass, DistinguishedName | Export-Csv -Path "C:\Audit\DomainAdmins.csv" -NoTypeInformation
Exercise 3: Find all computers running Windows Server 2019 or newer
Solution:
Get-ADComputer -Filter "OperatingSystem -like '*2019*' -or OperatingSystem -like '*2022*'" -Properties OperatingSystem | Select-Object Name, OperatingSystem
Exercise 4: Find accounts with SPNs that haven't had password changes in over 90 days (Kerberoasting risk)
Solution:
$oldDate = (Get-Date).AddDays(-90)
Get-ADUser -Filter {ServicePrincipalName -like "*" -and PasswordLastSet -lt $oldDate} -Properties ServicePrincipalName, PasswordLastSet | Select-Object Name, ServicePrincipalName, PasswordLastSet

🎓 Key Takeaways (Active Directory Security)

  • ✅ Import-Module ActiveDirectory first — RSAT must be installed
  • ✅ Search-ADAccount → Find inactive, locked-out, expired accounts
  • ✅ Get-ADGroupMember -Identity "Domain Admins" → Audit privileged access weekly
  • ✅ Accounts with PasswordNeverExpires are security violations — remediate immediately
  • ✅ KRBTGT password must be changed regularly (protect against Golden Ticket attacks)
  • ✅ Monitor for Kerberoasting — accounts with old passwords + SPNs are high risk
  • ✅ Export audits to CSV for compliance documentation
