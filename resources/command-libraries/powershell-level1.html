📂 PowerShell Level 1: Essential Cmdlets

Master the fundamentals of PowerShell. These are the cmdlets every Windows administrator and security professional must know — navigation, file operations, and getting help.

💡 Prerequisite: None — this is your starting point! PowerShell is available on Windows, Linux, and macOS (PowerShell Core).

📂 Navigation Cmdlets (Moving Around)

CmdletSyntaxDescriptionExample
Get-Location Get-Location Print current directory (like Linux `pwd`) Get-Location → C:\Users\student
Set-Location Set-Location [path] (alias: cd) Change to a different directory Set-Location C:\Windows
Set-Location .. (up one level)
Set-Location ~ (home directory)
Get-ChildItem Get-ChildItem [path] [-Recurse] (alias: dir, ls) List files and directories (like Linux `ls`) Get-ChildItem -Recurse (all files in all subdirectories)
Get-ChildItem *.log (only .log files)
📋 PowerShell Example:
PS C:\Users\student> Set-Location C:\Windows\System32
PS C:\Windows\System32> Get-Location
C:\Windows\System32
PS C:\Windows\System32> Get-ChildItem *.dll | Select-Object -First 5

📝 File Operations (Create, Copy, Move, Delete)

CmdletSyntaxDescriptionExample
New-Item New-Item -Path [path] -ItemType [File|Directory] Create a new file or directory New-Item -Path .\notes.txt -ItemType File
New-Item -Path .\Projects -ItemType Directory
Copy-Item Copy-Item -Path [source] -Destination [dest] (alias: cp) Copy files or directories Copy-Item .\notes.txt .\backup.txt
Copy-Item .\Projects -Destination .\Projects_Backup -Recurse
Move-Item Move-Item -Path [source] -Destination [dest] (alias: mv) Move or rename files/directories Move-Item .\notes.txt .\Projects\
Move-Item .\oldname.txt .\newname.txt (rename)
Remove-Item Remove-Item -Path [path] [-Recurse] (alias: rm, del) Delete files or directories (⚠️ IRREVERSIBLE!) Remove-Item .\notes.txt
Remove-Item .\Projects -Recurse -Force (⚠️ deletes everything)
⚠️ PowerShell Warning: Remove-Item -Recurse -Force deletes everything instantly with NO confirmation. Always double-check your path!

👁️ Viewing File Contents

CmdletSyntaxDescriptionExample
Get-Content Get-Content [file] [-Tail N] (alias: cat, type) Display file contents (use -Tail for last N lines) Get-Content C:\Windows\win.ini
Get-Content .\log.txt -Tail 50 -Wait (live monitoring)
💡 PowerShell Pro Tip: Get-Content -Wait is like Linux tail -f — watch logs in real-time!

📚 Getting Help (The Most Important Cmdlets!)

CmdletSyntaxDescriptionExample
Get-Command Get-Command [-Noun noun] [-Verb verb] Find available cmdlets Get-Command -Noun Process (find all process-related cmdlets)
Get-Command -Verb Get (find all "Get" cmdlets)
Get-Help Get-Help [cmdlet] [-Examples] [-Online] Display detailed help for any cmdlet Get-Help Get-Process
Get-Help Get-Process -Examples
Get-Help Get-Process -Online (opens browser)
Get-Member [object] | Get-Member Discover what properties/methods an object has Get-Process | Get-Member (see all properties of a process object)
Get-Service | Get-Member -MemberType Method
💡 PowerShell Pro Tip: Get-Command *process* finds any cmdlet with "process" in its name — wildcards work anywhere!

📝 Practice Exercises

Exercise 1: Navigate to C:\Windows\System32, list all .dll files, then count them
Solution:
Set-Location C:\Windows\System32
Get-ChildItem *.dll | Measure-Object | Select-Object -ExpandProperty Count
Exercise 2: Create a directory called "SecurityLab", then create a file "notes.txt" inside it
Solution:
New-Item -Path .\SecurityLab -ItemType Directory
New-Item -Path .\SecurityLab\notes.txt -ItemType File
Set-Content -Path .\SecurityLab\notes.txt -Value "Security lab notes"
Exercise 3: Find all cmdlets that work with "service" objects
Solution:
Get-Command -Noun Service

🎓 Key Takeaways (PowerShell Level 1)

  • ✅ Get-Location → Where am I? (pwd equivalent)
  • ✅ Set-Location → Move there (cd equivalent)
  • ✅ Get-ChildItem → What's here? (ls, dir equivalent)
  • ✅ New-Item, Copy-Item, Move-Item, Remove-Item → File operations
  • ✅ Get-Content → View files (cat, type equivalent)
  • ✅ Get-Command → Find cmdlets (your best friend!)
  • ✅ Get-Help → Learn any cmdlet (man equivalent)
  • ✅ Get-Member → Discover object properties (PowerShell superpower!)
