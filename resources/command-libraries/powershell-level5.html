🤖 PowerShell Level 5: Scripting & Automation

The final level — turn commands into powerful automation tools. Learn variables, loops, conditionals, error handling, functions, and remote management. This is what separates beginners from automation engineers.

💡 Prerequisite: PowerShell Levels 1-4. This level builds on everything you've learned.

📊 PowerShell vs Linux: Automation Concepts Comparison

ConceptPowerShellLinux EquivalentDescription
Variables $var = "value" var="value" (bash) Store data in memory for later use
Arrays @(1,2,3) (1 2 3) (bash) List/collection of items
ForEach Loop ForEach-Object { } for item in list; do done Iterate through each item and process it
Conditional if (condition) { } if [ condition ]; then fi Make decisions in code
Error Handling try { } catch { } trap { } or set -e Gracefully handle errors without crashing
Remote Execution Invoke-Command -ComputerName ssh user@host 'command' Run commands on remote computers
💡 Note: Both PowerShell and Linux have powerful automation capabilities. PowerShell excels at Windows management (AD, Registry, WMI). Linux (bash/ansible) excels at server automation. Many professionals learn both!

📦 Variables (Storing Data)

ConceptSyntaxDescriptionExample
String variable $name = "value" Store text $server = "SRV-FILE-01"
Number variable $num = 42 Store numbers $count = (Get-Process).Count
Array variable $list = @(item1, item2) Store a list of items $servers = @("SRV-01", "SRV-02", "SRV-03")
📋 Using variables in commands:
$computers = Get-ADComputer -Filter * | Select-Object -ExpandProperty Name
foreach ($computer in $computers) { Write-Host "Checking $computer" }

🔄 Loops — ForEach-Object (Process Each Item)

Syntax 1: Pipeline (most common)
Get-Process | ForEach-Object { $_.CPU }
Syntax 2: ForEach statement (clearer for complex scripts)
foreach ($process in Get-Process) { Write-Host $process.Name }
Real-world: Check multiple servers via Invoke-Command
$servers = @("SRV-01", "SRV-02", "SRV-03")
foreach ($server in $servers) {
    Invoke-Command -ComputerName $server -ScriptBlock { Get-Service "Spooler" }
}
💡 $_ (dollar underscore) represents the current object in the pipeline. Think of it as "this item" or "each item".

🔍 Where-Object — Filtering Data

Syntax 1: Script block (traditional)
Get-Process | Where-Object { $_.CPU -gt 100 }
Syntax 2: Comparison statement (PowerShell 7+, cleaner)
Get-Process | Where-Object CPU -gt 100
Real-world: Find suspicious processes
Get-Process | Where-Object { $_.Path -like "*\Temp\*" -or $_.Path -like "*\Users\*\Downloads\*" } | Select-Object Name, Id, Path
💡 Common operators: `-eq` (equals), `-ne` (not equals), `-gt` (greater than), `-lt` (less than), `-like` (wildcard match), `-match` (regex match)

⚖️ If/Else — Making Decisions

Basic If statement:
if ($cpuUsage -gt 90) {
    Write-Warning "High CPU usage detected!"
}
If/Else:
if ($processName -eq "powershell") {
    Write-Host "Found PowerShell"
} else {
    Write-Host "Not PowerShell"
}
Real-world: Server health check
$service = Get-Service -Name "Spooler" -ErrorAction SilentlyContinue
if ($service.Status -eq "Running") {
    Write-Host "Spooler is running" -ForegroundColor Green
} else {
    Write-Warning "Spooler is stopped!"
}

🛡️ Try/Catch — Handle Errors Gracefully

Basic error handling:
try {
    Get-Service -Name "NonexistentService" -ErrorAction Stop
} catch {
    Write-Warning "Service not found: $($_.Exception.Message)"
}
Real-world: Remote command with fallback
$computers = @("SRV-01", "SRV-02")
foreach ($computer in $computers) {
    try {
        Invoke-Command -ComputerName $computer -ScriptBlock { Get-Service } -ErrorAction Stop
    } catch {
        Write-Warning "Failed to connect to $computer"
    }
}
💡 Critical: Always use `-ErrorAction Stop` in try blocks, otherwise non-terminating errors won't trigger catch.

📦 Functions — Build Your Own Commands

Simple function:
function Get-ServerUptime {
    param($ComputerName)
    Get-CimInstance -ClassName Win32_OperatingSystem -ComputerName $ComputerName | Select-Object LastBootUpTime
}
Get-ServerUptime -ComputerName "localhost"
Function with advanced parameters:
function Test-ServerConnection {
    param(
        [Parameter(Mandatory=$true)]
        [string[]]$ComputerName,
        [int]$Port = 445
    )
    foreach ($computer in $ComputerName) {
        $result = Test-NetConnection -ComputerName $computer -Port $Port -WarningAction SilentlyContinue
        if ($result.TcpTestSucceeded) { Write-Host "$computer is reachable on port $Port" -ForegroundColor Green }
        else { Write-Warning "$computer is NOT reachable on port $Port" }
    }
}
Test-ServerConnection -ComputerName "SRV-01", "SRV-02" -Port 3389

🌐 Invoke-Command — Manage Your Entire Network

This is the most powerful cmdlet for administrators. Run commands on any computer in your domain without leaving your chair.

Run a single command on one remote computer:
Invoke-Command -ComputerName "SRV-FILE-01" -ScriptBlock { Get-Service }
Run the same command on multiple computers:
$servers = @("SRV-01", "SRV-02", "SRV-03")
Invoke-Command -ComputerName $servers -ScriptBlock { Get-Service -Name "Spooler" | Select-Object MachineName, Status }
Real-world: Check all domain controllers for replication issues
$DCs = Get-ADDomainController -Filter * | Select-Object -ExpandProperty Name
Invoke-Command -ComputerName $DCs -ScriptBlock { repadmin /replsum }
Compare Linux SSH equivalent:
# PowerShell:
Invoke-Command -ComputerName "SRV-01" -ScriptBlock { Get-Process }

# Linux equivalent (SSH):
ssh user@server "ps aux"

# Linux loop for multiple servers:
for server in srv01 srv02 srv03; do ssh $server "uptime"; done
💡 Prerequisite: WinRM (Windows Remote Management) must be enabled. Run Enable-PSRemoting -Force as admin on target computers.

📜 Real-World Automation Scripts (Copy-Paste Ready)

🔍 Script 1: Audit all servers for running services
$servers = Get-ADComputer -Filter {OperatingSystem -like "*Server*"} | Select-Object -ExpandProperty Name
$results = @()
foreach ($server in $servers) {
    try {
        $services = Invoke-Command -ComputerName $server -ScriptBlock { Get-Service | Where-Object {$_.Status -eq "Running"} } -ErrorAction Stop
        $results += [PSCustomObject]@{ Server = $server; RunningServices = $services.Count }
    } catch {
        $results += [PSCustomObject]@{ Server = $server; RunningServices = "Unreachable" }
    }
}
$results | Export-Csv -Path "C:\Audit\ServerServices.csv" -NoTypeInformation
🛡️ Script 2: Find inactive AD users and export report
$inactiveUsers = Search-ADAccount -AccountInactive -TimeSpan 90.00:00:00
$report = $inactiveUsers | Select-Object Name, SamAccountName, LastLogonDate, Enabled
$report | Export-Csv -Path "C:\Audit\InactiveUsers.csv" -NoTypeInformation
Write-Host "Found $($inactiveUsers.Count) inactive users" -ForegroundColor Yellow
🔥 Script 3: Monitor for suspicious processes across the network
$suspiciousProcesses = @("nc.exe", "ncat.exe", "mimikatz.exe", "procdump.exe")
$servers = Get-ADComputer -Filter * | Select-Object -ExpandProperty Name
$alerts = @()
foreach ($server in $servers) {
    $processes = Invoke-Command -ComputerName $server -ScriptBlock { Get-Process | Select-Object Name, Id, Path } -ErrorAction SilentlyContinue
    foreach ($proc in $suspiciousProcesses) {
        $found = $processes | Where-Object { $_.Name -eq $proc }
        if ($found) { $alerts += [PSCustomObject]@{ Server = $server; Process = $proc; Path = $found.Path } }
    }
}
$alerts | Export-Csv -Path "C:\Audit\SuspiciousProcesses.csv" -NoTypeInformation

📝 Practice Exercises

Exercise 1: Write a script that gets all running processes on your computer, filters those using over 100 MB of memory, and writes them to a CSV file
Solution:
$processes = Get-Process | Where-Object { $_.WorkingSet -gt 100MB }
$processes | Select-Object Name, Id, @{Name="Memory(MB)";Expression={[math]::Round($_.WorkingSet/1MB,2)}} | Export-Csv -Path "HighMemoryProcesses.csv" -NoTypeInformation
Exercise 2: Write a function that takes a computer name and a service name, checks if the service is running, and returns "Running" or "Stopped"
Solution:
function Get-ServiceStatus {
    param([string]$ComputerName, [string]$ServiceName)
    $service = Get-Service -Name $ServiceName -ComputerName $ComputerName -ErrorAction SilentlyContinue
    if ($service) { return $service.Status } else { return "Not Found" }
}
Exercise 3: Write a script that checks disk space on all domain controllers and alerts if any have less than 10% free space
Solution:
$DCs = Get-ADDomainController -Filter * | Select-Object -ExpandProperty Name
foreach ($DC in $DCs) {
    $disk = Invoke-Command -ComputerName $DC -ScriptBlock { Get-PSDrive -Name C }
    $freePercent = ($disk.Free / $disk.Used) * 100
    if ($freePercent -lt 10) { Write-Warning "$DC has only $([math]::Round($freePercent,2))% free space on C:" }
}
