🐧 Linux Level 1: Essential Commands

Master the fundamentals. These are the commands every Linux user must know — navigation, file operations, viewing files, and basic process management.

💡 Prerequisite: None — this is your starting point!

📂 Navigation Commands (Moving Around)

CommandSyntaxDescriptionExample
pwd pwd Print Working Directory — shows your current location pwd → /home/student
ls ls [options] [path] List files and directories ls -la (list all, including hidden, with details)
cd cd [path] Change Directory — move to another folder cd /var/log → cd .. (up one level) → cd ~ (home)
📋 Real-world example:
student@linux:~$ pwd
/home/student
student@linux:~$ ls -la
total 36
drwxr-x--- 5 student student 4096 May 13 10:30 .
drwxr-xr-x 3 root root 4096 May 13 10:28 ..
-rw-r--r-- 1 student student 220 May 13 10:28 .bashrc
student@linux:~$ cd /etc
student@linux:/etc$ pwd
/etc

📝 File Operations (Create, Copy, Move, Delete)

CommandSyntaxDescriptionExample
touch touch [filename] Create an empty file or update timestamp touch notes.txt → Creates empty file "notes.txt"
cp cp [source] [destination] Copy files or directories cp notes.txt notes_backup.txt → Copies "notes.txt" to a new file "notes_backup.txt" (original unchanged)
cp -r folder1 folder2 → Copies entire folder1 (including all contents) into folder2. If folder2 doesn't exist, it's created. If it exists, folder1 goes inside folder2.
mv mv [source] [destination] Move or rename files/directories mv notes.txt documents/ → Moves "notes.txt" into the "documents" folder (original no longer in current location)
mv oldname.txt newname.txt → Renames the file within the same directory
rm rm [options] [file] Remove (delete) files — ⚠️ IRREVERSIBLE! rm notes.txt → Deletes "notes.txt" permanently
rm -rf directory/ → Deletes entire directory and ALL contents (⚠️ extremely dangerous!)
mkdir mkdir [directory] Make Directory — create a new folder mkdir projects → Creates folder named "projects" in current location
mkdir -p projects/2024/logs → Creates parent directories "projects" and "2024" if they don't exist, then "logs" inside
rmdir rmdir [directory] Remove empty directory rmdir empty_folder → Deletes ONLY if folder is completely empty
rm -rf non_empty_folder → Use this for non-empty folders (dangerous!)

📖 Understanding cp, mv, and rm (Detailed Examples)

🔍 cp (copy) - Original stays, new copy created:
cp file.txt backup.txt
→ Creates "backup.txt" that is identical to "file.txt"
→ "file.txt" still exists. You now have TWO files.

cp -r folder1 folder2
→ If "folder2" does NOT exist: Creates "folder2" and copies ALL contents of "folder1" into it.
→ If "folder2" already exists: Copies "folder1" INSIDE "folder2" (result: folder2/folder1/...)
Use case: Backing up a project before making changes.
🔍 mv (move/rename) - Original moves, no copy:
mv file.txt backup.txt
→ Renames "file.txt" to "backup.txt". Same file, new name. Still ONE file.

mv file.txt documents/
→ Moves "file.txt" INTO the "documents" folder.
→ The file is no longer in the original location.

mv documents/ /home/student/backup/
→ Moves the ENTIRE "documents" folder to a new location.
Use case: Organizing files, renaming directories.
⚠️ rm (remove) - FOREVER DELETE (no undo, no recycle bin!):
rm file.txt → Deletes "file.txt" permanently.
rm -r folder/ → Deletes the folder and EVERYTHING inside it.
rm -rf folder/ → Force delete (ignores warnings, even if write-protected).
🔴 CRITICAL: There is NO UNDO. Always double-check before hitting Enter!

👁️ Viewing Files (Read, Not Edit)

CommandSyntaxDescriptionExample
cat cat [file] Concatenate and display entire file (good for small files) cat /etc/hostname
less less [file] View file page by page (scroll with arrows, press 'q' to quit) less /var/log/syslog
head head -n [file] Show first N lines (default 10) head -20 /var/log/auth.log
tail tail -n [file] Show last N lines (default 10) tail -f /var/log/syslog (follow live updates)
📋 Live log monitoring (critical for security!):
sudo tail -f /var/log/auth.log
# Watch failed SSH login attempts in real-time

⚙️ Process Management (What's Running?)

CommandSyntaxDescriptionExample
ps ps [options] Show running processes (snapshot) ps aux (all users, detailed) → ps aux | grep nginx
top top Real-time process viewer (press 'q' to quit) top → then 'P' (sort by CPU), 'M' (sort by memory)
kill kill [signal] [PID] Send signal to process (default 15 = TERM) kill 1234 → kill -9 1234 (force kill)
pkill pkill [process-name] Kill by process name pkill firefox
💡 Finding a PID: pgrep -u root bash (find all bash processes running as root)

📝 Practice Exercises (Try These!)

Exercise 1: Navigate to /var/log, list all files, then view auth.log with less
Solution:
cd /var/log
ls -la
less auth.log
# Press 'q' to exit less
Exercise 2: Create a directory called "practice", create a file "test.txt", copy it to "test_backup.txt"
Solution:
mkdir practice
cd practice
touch test.txt
cp test.txt test_backup.txt
ls -la # Verify both files exist
Exercise 3: Find the PID of a running process (e.g., systemd) and check its details with ps
Solution:
pgrep systemd
ps aux | grep systemd
# Or: ps -p [PID] -f

🎓 Key Takeaways (Level 1 Essentials)

  • ✅ pwd → Where am I?
  • ✅ ls → What's here?
  • ✅ cd → Move there
  • ✅ cp, mv, rm → Copy, move, delete (⚠️ rm is irreversible!)
  • ✅ cat, less, head, tail → View files safely
  • ✅ ps, top, kill → Manage processes
  • ✅ man, --help → Your best friends when stuck
