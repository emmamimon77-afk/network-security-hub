🛡️ PowerShell Level 2: Security & Incident Response

Essential cmdlets for security analysts, SOC teams, and incident responders. Learn to investigate processes, analyze event logs, detect network connections, and respond to security incidents.

💡 Prerequisite: PowerShell Level 1 (navigation, files, help). If you're new, start there first!

🔍 Process Investigation (Find Suspicious Activity)

CmdletSyntaxDescriptionExample & Explanation
Get-Process Get-Process [[-Name] processname] Lists all running processes with CPU, memory, and ID Get-Process | Sort-Object CPU -Descending | Select -First 10
→ Shows top 10 CPU-consuming processes (detect cryptominers!)
Get-Process -Name "powershell" | Select-Object Id, CPU, WorkingSet
→ Shows specific details about PowerShell processes
Stop-Process Stop-Process -Id [PID] [-Force] Terminates a running process (use with caution) Stop-Process -Id 1234 -Force
→ Immediately terminates process with ID 1234
⚠️ Warning: Can crash the system if used on critical processes!
💡 Detect suspicious processes:
Get-Process | Where-Object {$_.Path -like "*\Temp\*"}
→ Finds processes running from Temp folder (common malware location)

📜 Event Log Analysis (Track Attacks & Failures)

Critical Windows Event IDs for Security

Event IDDescriptionWhat It Means
4624 Successful logon Normal user login. Monitor for unusual hours or location
4625 Failed logon Brute-force indicator — too many = active attack
4648 Logon using explicit credentials Possible lateral movement (runas, scheduled tasks)
4672 Special privileges assigned to new logon Admin logon — monitor carefully
4688 New process created Detect malware execution (with command-line logging enabled)
7045 New service installed Persistence mechanism — malware loves this

Querying Event Logs with PowerShell

📋 Get failed logons in last 24 hours:
$startTime = (Get-Date).AddHours(-24)
Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4625; StartTime=$startTime} | Select-Object TimeCreated, Message
📋 Get process creation events (requires command-line auditing enabled):
Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4688} | Select-Object -First 20 | Format-List TimeCreated, Message
💡 Pro tip: Enable command-line auditing in Event 4688 via Group Policy to see exactly what commands were run!

🌐 Network Investigation (Find Active Connections)

CmdletSyntaxDescriptionExample & Explanation
Get-NetTCPConnection Get-NetTCPConnection [-State Listen\|Established] Shows TCP connections, listening ports, and process IDs Get-NetTCPConnection -State Listen
→ Shows all ports your machine is listening on (potential backdoors!)
Get-NetTCPConnection -State Established | Where-Object {$_.RemotePort -eq 4444}
→ Find connections on suspicious port 4444 (common for reverse shells)
🚨 Incident Response Quick Check:
Get-NetTCPConnection -State Established | Group-Object RemotePort | Sort-Object Count -Descending | Select-Object -First 10
→ Shows top 10 remote ports your machine is connected to — look for unusual ones like 4444, 1337, 31337!

🔐 File Hashing (Verify Integrity, Detect Tampering)

CmdletSyntaxDescriptionExample & Explanation
Get-FileHash Get-FileHash -Path [file] -Algorithm [SHA256\|MD5] Generates cryptographic hash of a file (fingerprint) Get-FileHash -Path C:\Windows\System32\notepad.exe -Algorithm SHA256
→ Returns a unique hash like a1b2c3d4...
Use case: Compare against known good hash to detect if system file was replaced by rootkit.

🚨 Incident Response Quick Checklist (Copy-Paste Ready)

Step 1 — Check for suspicious processes (CPU hogs):
Get-Process | Sort-Object CPU -Descending | Select-Object -First 10

Step 2 — Find processes from suspicious locations:
Get-Process | Where-Object {$_.Path -like "*\Users\*\AppData\*"} | Select-Object Name, Id, Path

Step 3 — Check for listening ports (potential backdoors):
Get-NetTCPConnection -State Listen | Select-Object LocalPort, OwningProcess | ForEach-Object { $_.OwningProcess = (Get-Process -Id $_.OwningProcess).ProcessName; $_ }

Step 4 — Review failed logins in last hour:
$startTime = (Get-Date).AddHours(-1); Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4625; StartTime=$startTime} | Select-Object TimeCreated, Message

Step 5 — Check for suspicious scheduled tasks (persistence):
Get-ScheduledTask | Where-Object {$_.State -ne "Disabled"} | Select-Object TaskName, TaskPath, State

📝 Practice Exercises

Exercise 1: Find all processes using more than 100 MB of memory
Solution:
Get-Process | Where-Object {$_.WorkingSet -gt 100MB} | Select-Object Name, Id, @{Name="Memory(MB)";Expression={[math]::Round($_.WorkingSet/1MB,2)}}
Exercise 2: Find all failed logins (Event ID 4625) from a specific IP address
Solution:
Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4625} | Where-Object {$_.Message -like "*192.168.1.100*"} | Select-Object TimeCreated, Message
Exercise 3: Generate SHA256 hash of a file and compare to a known good hash
Solution:
$hash = (Get-FileHash -Path C:\file.exe -Algorithm SHA256).Hash
if ($hash -eq "known-good-hash-here") { Write-Host "File is clean" } else { Write-Host "File may be tampered!" }

🎓 Key Takeaways (PowerShell Security & IR)

  • ✅ Get-Process → Detect cryptominers and suspicious processes
  • ✅ Get-WinEvent → Query security logs (Event ID 4625 = failed logins)
  • ✅ Get-NetTCPConnection → Find listening ports (potential backdoors)
  • ✅ Get-FileHash → Verify file integrity (detect tampering)
  • ✅ Enable command-line auditing — see exactly what attackers ran
  • ✅ Always investigate Event ID 4688 (process creation) — reveals malware execution
