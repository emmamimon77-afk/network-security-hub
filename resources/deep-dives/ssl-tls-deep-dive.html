🔐 SSL/TLS Deep Dive

Secure Sockets Layer / Transport Layer Security is the foundation of internet security — HTTPS, secure email, VPNs, and more. This deep dive covers certificate authorities, the TLS handshake, cipher suites, Perfect Forward Secrecy, and common attacks.

💡 Target audience: Anyone who wants to understand how encryption works on the web, and how to configure it securely.

📜 SSL vs TLS — What's the Difference?

  • SSL (Secure Sockets Layer): Developed by Netscape in 1995. SSLv2 and SSLv3 are now deprecated and insecure (POODLE attack).
  • TLS (Transport Layer Security): Successor to SSL. TLS 1.0 (1999), TLS 1.1 (2006), TLS 1.2 (2008), TLS 1.3 (2018).
  • Today: Only TLS 1.2 and TLS 1.3 are considered secure. TLS 1.3 is the current standard.
⚠️ Never use SSLv2, SSLv3, or TLS 1.0. They are vulnerable to POODLE, BEAST, and other attacks.

🏛️ PKI — Public Key Infrastructure

PKI is the system that enables secure communication over untrusted networks. It consists of:

  • Certificate Authority (CA): Trusted third party that issues digital certificates (e.g., Let's Encrypt, DigiCert, GlobalSign)
  • Registration Authority (RA): Verifies the identity of certificate requesters
  • Certificate Database: Stores issued certificates and CRLs (Certificate Revocation Lists)
  • Certificate Revocation List (CRL) / OCSP: Methods to check if a certificate is still valid

📜 Digital Certificates — X.509

An X.509 certificate contains:

  • Subject (who the certificate belongs to — domain name or organization)
  • Issuer (which CA issued it)
  • Public key
  • Validity period (not before / not after)
  • Signature algorithm (e.g., sha256RSA)
  • Extensions (Subject Alternative Names, Key Usage, etc.)

Types of Certificates:

TypeValidation LevelUse Case
Domain Validation (DV) Domain ownership only Personal websites, blogs (Let's Encrypt)
Organization Validation (OV) Domain + organization identity Business websites
Extended Validation (EV) Rigorous identity verification Banks, e-commerce (green bar historically)

View Certificate Details (OpenSSL):

# Check a website's certificate openssl s_client -connect google.com:443 -showcerts
# View certificate details echo | openssl s_client -connect google.com:443 2>/dev/null | openssl x509 -text

💡 Key Insight: The pre-master secret is generated by the client and encrypted with the server's public key. Only the server can decrypt it using its private key. Both sides then derive the same session keys.

🔢 Cipher Suites — The Encryption Recipe

A cipher suite defines four algorithms:

  • Key Exchange: RSA, ECDHE, DHE
  • Authentication: RSA, ECDSA
  • Bulk Encryption: AES, ChaCha20
  • Message Authentication Code (MAC): SHA256, SHA384
Example cipher suite: TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256
Breakdown:
  • ECDHE = Key exchange (Elliptic Curve Diffie-Hellman Ephemeral)
  • RSA = Authentication (server cert signed with RSA)
  • AES_128_GCM = Bulk encryption (AES-128 in Galois/Counter Mode)
  • SHA256 = MAC/hashing
💡 Perfect Forward Secrecy (PFS): Key exchange using Diffie-Hellman Ephemeral (DHE or ECDHE). If the server's private key is compromised later, past sessions cannot be decrypted.

⚡ TLS 1.3 — Faster and More Secure

FeatureTLS 1.2TLS 1.3
Handshake time 2 RTT (2 round trips) 1 RTT (or 0-RTT for resumption)
Legacy algorithms removed Supports MD5, SHA1, RC4, 3DES Only secure AEAD ciphers (AES-GCM, ChaCha20-Poly1305)
Perfect Forward Secrecy Optional (requires DHE/ECDHE) Mandatory (all key exchanges provide PFS)

⚔️ Common SSL/TLS Attacks

  • Heartbleed (CVE-2014-0160): Memory leak in OpenSSL. Allowed attackers to read server memory (including private keys). Fixed in 2014.
  • POODLE (CVE-2014-3566): Downgrade attack forcing SSLv3. Mitigation: Disable SSLv3.
  • BEAST (CVE-2011-3389): Chosen-plaintext attack against TLS 1.0. Mitigation: Use TLS 1.2+.
  • CRIME (CVE-2012-4929): Compression side-channel. Mitigation: Disable TLS compression.
  • Logjam (CVE-2015-4000): Downgrade attack to 512-bit Diffie-Hellman. Mitigation: Use 2048-bit+ DH parameters.
⚠️ Modern best practices: Use TLS 1.2+ only, disable weak cipher suites, enable HSTS, use strong DH parameters.

🛠️ Practical OpenSSL Commands

Generate a Private Key and Self-Signed Certificate:

# Generate private key (2048-bit RSA) openssl genrsa -out private.key 2048
# Generate self-signed certificate (valid for 365 days) openssl req -new -x509 -key private.key -out certificate.crt -days 365

Generate a Certificate Signing Request (CSR):

# Generate CSR for a CA to sign openssl req -new -key private.key -out request.csr

Check a Website's TLS Configuration:

# Check which cipher suites are supported nmap --script ssl-enum-ciphers -p 443 example.com
# Using openssl (specific cipher) openssl s_client -connect example.com:443 -cipher 'ECDHE-RSA-AES128-GCM-SHA256'

Convert Certificate Formats:

# PEM to DER (binary) openssl x509 -in cert.pem -outform DER -out cert.der
# DER to PEM openssl x509 -in cert.der -inform DER -outform PEM -out cert.pem
# PKCS#12 (PFX) to PEM (common for importing into applications) openssl pkcs12 -in keystore.pfx -out keystore.pem -nodes

Check Certificate Expiration:

openssl x509 -in certificate.crt -noout -dates

🛡️ HSTS — Force HTTPS Only

HSTS tells browsers to always use HTTPS for your domain, preventing SSL stripping attacks.

# Add to web server response header Strict-Transport-Security: max-age=31536000; includeSubDomains; preload
  • max-age=31536000 — 1 year validity
  • includeSubDomains — Apply to all subdomains
  • preload — Submit to browser preload list

📝 Practice Exercises

Exercise 1: Generate a self-signed certificate and key, then start a temporary HTTPS server with Python to test it
Solution:
openssl req -x509 -newkey rsa:2048 -keyout key.pem -out cert.pem -days 365 -nodes
python3 -m http.server --ssl --cert cert.pem --key key.pem 8443
Then visit https://localhost:8443 (accept the warning).
Exercise 2: Verify which TLS version and cipher suite a website uses
Solution:
openssl s_client -connect google.com:443 -tls1_2
Look for "Cipher" in the output.
Exercise 3: Check your own website's TLS configuration using ssllabs.com (online)
Solution:
Go to https://www.ssllabs.com/ssltest/ and enter your domain.

🎓 Key Takeaways (SSL/TLS)

  • ✅ SSL is dead; only use TLS 1.2 and TLS 1.3
  • ✅ PKI — Certificate Authorities establish trust
  • ✅ X.509 certificates contain public key, identity, and CA signature
  • ✅ TLS handshake — Client hello → Server hello → Certificate → Key exchange → Finished
  • ✅ Perfect Forward Secrecy (PFS) — Use DHE/ECDHE to protect past sessions
  • ✅ TLS 1.3 — Faster, only secure ciphers, mandatory PFS
  • ✅ HSTS — Force HTTPS to prevent SSL stripping
  • ✅ OpenSSL is the Swiss Army knife for certificate operations
