🪟 Windows Security Internals — Deep Dive

Understand the security features built into Windows. This deep dive covers Credential Guard, PatchGuard, Secure Kernel, Virtualization-Based Security (VBS), advanced event log analysis, and Windows Defender internals.

💡 Target audience: Windows administrators, security professionals, and anyone defending Windows environments.

🏛️ Windows Security Architecture Overview

🔒 Virtualization-Based Security (VBS) — The Foundation

VBS uses hardware virtualization to create a secure, isolated region of memory separate from the normal OS. This "Virtual Secure Mode" (VSM) protects critical security components even if the kernel is compromised.

┌─────────────────────────────────────────────────────────────┐ │ Normal OS (Ring 0) │ │ ┌─────────────────────────────────────────────────────┐ │ │ │ Hyper-V Hypervisor │ │ │ └─────────────────────────────────────────────────────┘ │ │ │ │ ┌─────────────────────┐ ┌─────────────────────────┐ │ │ │ Normal Kernel │ │ Virtual Secure Mode │ │ │ │ (Ring 0) │ │ (Ring -1) │ │ │ │ │ │ ┌───────────────────┐ │ │ │ │ │ │ │ Credential Guard │ │ │ │ │ │ │ ├───────────────────┤ │ │ │ │ │ │ │ Secure Kernel │ │ │ │ │ │ │ ├───────────────────┤ │ │ │ │ │ │ │ Isolated User │ │ │ │ │ │ │ │ Mode │ │ │ │ │ │ │ └───────────────────┘ │ │ │ └─────────────────────┘ └─────────────────────────┘ │ └─────────────────────────────────────────────────────────────┘

Check VBS Status:

# PowerShell (Admin) Get-ComputerInfo -Property "DeviceGuard*"
# Or via System Information msinfo32 (look for "Virtualization-based security")

🔐 Credential Guard — Protect NTLM Hashes & Kerberos Tickets

Credential Guard uses VBS to isolate LSASS (Local Security Authority Subsystem Service). Even if an attacker compromises the kernel, they cannot extract NTLM hashes or Kerberos tickets.

Without Credential Guard: With Credential Guard: ┌─────────────┐ ┌─────────────┐ │ LSASS │ ◀── Attacker │ LSASS │ (Isolated) │ (NTLM Hash) │ can read │ (NTLM Hash) │ └─────────────┘ └──────┬──────┘ │ ┌────┴────┐ │ VBS │ (Protected) │ Memory │ └─────────┘

Enable Credential Guard:

# Via Group Policy Computer Configuration → Administrative Templates → System → Device Guard → Turn on Virtualization Based Security
# Set to "Enabled with UEFI lock"
# Select "Credential Guard" under "Select Platform Security Level"
💡 Requirement: UEFI, Secure Boot, and virtualization extensions (VT-x/AMD-V) must be enabled.

🛡️ PatchGuard — Kernel Protection

PatchGuard prevents unauthorized modifications to the Windows kernel. It detects and blocks rootkits that try to hook critical kernel structures.

What PatchGuard Protects:

  • ✅ System Service Dispatch Table (SSDT) — System call hooks
  • ✅ Interrupt Descriptor Table (IDT) — Interrupt handlers
  • ✅ Global Descriptor Table (GDT) — Memory segments
  • ✅ Kernel code sections
  • ✅ Critical system structures
⚠️ Note: PatchGuard cannot be disabled on 64-bit Windows. It's why rootkits are rarer on modern Windows than on Linux.

🔐 Secure Kernel & HVCI (Hypervisor-Protected Code Integrity)

HVCI ensures that only signed, verified code can run in kernel mode. It uses the hypervisor to enforce code integrity policies.

Check HVCI Status:

# PowerShell (Admin) Get-ComputerInfo -Property "DeviceGuard*"
# Or via Windows Security Windows Security → Device Security → Core Isolation → Memory Integrity

Enable Memory Integrity (HVCI):

Windows Security → Device Security → Core Isolation details → Memory integrity → ON → Reboot
💡 Note: HVCI may conflict with some older drivers. Test before enabling on critical systems.

🦠 Windows Defender ATP — Next-Gen Antivirus

Modern Windows includes enterprise-grade EDR (Endpoint Detection and Response) capabilities.

Key Components:

  • Cloud-delivered protection: Real-time updates from Microsoft cloud
  • Behavioral monitoring: Detects ransomware, fileless malware
  • Attack Surface Reduction (ASR): Blocks common attack patterns
  • Network protection: Blocks malicious domains/IPs
  • Controlled folder access: Protects against ransomware

Check Defender Status (PowerShell):

Get-MpComputerStatus
# Check real-time protection (Get-MpPreference).DisableRealtimeMonitoring
# Update definitions Update-MpSignature
# Run a quick scan Start-MpScan -ScanType QuickScan

📋 Event Log Analysis — Security Events You Must Monitor

Event IDEvent NameDescriptionImportance
4624 An account was successfully logged on Successful authentication (normal, but spikes = suspicious) 🔴 Critical
4625 An account failed to log on Brute-force detection — too many = active attack 🔴 Critical
4672 Special privileges assigned to new logon Admin logon — monitor carefully 🟡 High
4688 A new process has been created Detect malware execution (requires command-line logging) 🟡 High
4698 A scheduled task was created Persistence mechanism — malware loves this 🔴 Critical
7045 A new service was installed Service persistence — common for backdoors 🔴 Critical

PowerShell Commands for Event Analysis:

# Get failed logons in last 24 hours $startTime = (Get-Date).AddHours(-24) Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4625; StartTime=$startTime} | Select-Object TimeCreated, Message
# Get admin logons (Event ID 4672) Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4672} -MaxEvents 50 | Select-Object TimeCreated, Message
# Monitor new processes (requires command-line logging) Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4688} -MaxEvents 50 | Format-List TimeCreated, Message
# Export to CSV for analysis Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4625} -MaxEvents 1000 | Export-Csv -Path "failed_logons.csv"

📝 Practice Exercises

Exercise 1: Check if VBS and Credential Guard are enabled on your Windows system
Solution (PowerShell as Admin):
Get-ComputerInfo -Property "DeviceGuard*"
Look for DeviceGuardSecurityServicesConfigured and DeviceGuardSecurityServicesRunning.
Exercise 2: Find all failed logins (Event ID 4625) from the last hour
Solution:
$startTime = (Get-Date).AddHours(-1)
Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4625; StartTime=$startTime} | Select-Object TimeCreated, Message
Exercise 3: Check if real-time protection is enabled on Windows Defender
Solution:
Get-MpPreference | Select-Object DisableRealtimeMonitoring
If False, real-time protection is ON. If True, it's OFF.
Exercise 4: Export all process creation events (4688) from the last week to CSV
Solution:
$startTime = (Get-Date).AddDays(-7)
Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4688; StartTime=$startTime} | Export-Csv -Path "process_creation_events.csv"

🎓 Key Takeaways (Windows Security Internals)

  • ✅ VBS (Virtualization-Based Security) — Creates isolated secure memory region
  • ✅ Credential Guard — Protects NTLM hashes and Kerberos tickets
  • ✅ PatchGuard — Prevents kernel modification (can't be disabled)
  • ✅ HVCI (Memory Integrity) — Ensures only signed kernel code runs
  • ✅ Event IDs to monitor: 4624 (success), 4625 (failure), 4672 (admin), 4688 (process), 4698 (task), 7045 (service)
  • ✅ Windows Defender ATP — Enterprise-grade EDR with ASR rules
  • ✅ Enable command-line auditing to see exact commands in Event 4688
