📡 Wireshark Mastery — Deep Dive

Wireshark is the world's most popular network protocol analyzer. This deep dive covers capture vs display filters, TCP analysis, following streams, statistical tools, and command-line magic with tshark.

💡 Target audience: Network engineers, security analysts, and anyone who wants to see what's really happening on the wire.

📥 Installation

# Ubuntu/Debian sudo apt update && sudo apt install wireshark -y # Fedora/RHEL sudo dnf install wireshark -y # macOS (Homebrew) brew install --cask wireshark # Windows # Download from https://www.wireshark.org/download.html
⚠️ During installation on Linux, you may be prompted to allow non-superusers to capture packets. Select "Yes" if you want to avoid running Wireshark with `sudo`.

🎭 Capture Filters vs. Display Filters

This is the most important distinction in Wireshark.

Feature Capture Filter Display Filter
When applied Before capture starts After capture (on saved packets)
Syntax `tcp port 80` (Berkeley Packet Filter) `tcp.port == 80` (Wireshark display filter)
Impact on data Dropped packets are gone forever Packets are hidden from view, but still in the file
📋 When to use which: - Capture filter: You're on a very busy network and need to reduce the capture file size. - Display filter: You want to analyze a full capture from different angles.

🎣 Common Capture Filters (BPF Syntax)

# Capture only web traffic (ports 80 and 443) tcp port 80 or tcp port 443 # Capture only traffic to/from a specific host host 192.168.1.100 # Capture traffic from a specific subnet src net 192.168.1.0/24 # Capture all traffic except SSH (port 22) not port 22

🔍 Essential Display Filters (Most Powerful Feature)

# Filter by protocol dns http tls icmp # Filter by IP address (source or destination) ip.addr == 192.168.1.1 # Filter by source IP ip.src == 192.168.1.10 # Filter by destination IP ip.dst == 8.8.8.8 # Filter by port tcp.port == 443 # Filter by TCP flags (SYN packets) tcp.flags.syn == 1 and tcp.flags.ack == 0 # Find retransmissions (network issues) tcp.analysis.retransmission # Find HTTP POST requests (login attempts) http.request.method == "POST"
💡 Pro tip: Right-click any field in a packet and select "Apply as Filter" to auto-generate the filter expression.

🧮 Logical Operators

Operator Alias Example
and && `ip.src == 10.0.0.1 and tcp.port == 80`
or || `ip.src == 10.0.0.1 or ip.src == 192.168.1.1`
not ! `not tcp.port == 22`
contains — `http contains "password"`
📋 Complex filter example: `(ip.src == 192.168.1.10 or ip.src == 10.0.0.5) and (tcp.port == 80 or tcp.port == 443) and not icmp`

💬 Following Streams — Reconstruct Conversations

This is pure magic for security analysts.

Right-click a packet → Follow → TCP Stream (or UDP Stream) Wireshark opens a window showing the entire conversation, removing protocol headers.
⚠️ You can find plaintext passwords in HTTP or FTP streams. This is why HTTPS is critical.

📊 Statistical Analysis

Wireshark's statistical tools help you identify traffic patterns and anomalies.

  • Statistics → Endpoints — See all IP/MAC addresses communicating.
  • Statistics → Conversations — Traffic between pairs of endpoints.
  • Statistics → Protocol Hierarchy — See which protocols dominate traffic.
  • Statistics → I/O Graph — Visualize traffic over time.

🖥️ tshark — The Command-Line Wireshark

Automate captures and analysis with the same filtering power.

# Capture 100 packets and write to file tshark -i eth0 -c 100 -w capture.pcap # Read a pcap and apply display filter tshark -r capture.pcap -Y "tcp.port == 443" # Show HTTP requests only tshark -r capture.pcap -Y "http.request" -T fields -e http.request.method -e http.request.uri # Get statistics on IP addresses tshark -r capture.pcap -z endpoints,ip

📋 Case Study: Diagnosing a Slow Web Server

Scenario: A web server is responding slowly. Capture traffic and open in Wireshark.

Step 1: Filter to web traffic: `tcp.port == 80 or tcp.port == 443`
Step 2: Check for TCP retransmissions: `tcp.analysis.retransmission`
Step 3: Follow TCP stream of a slow request to see delay between request and response.
Step 4: Use I/O Graph to see traffic spikes.

📝 Practice Exercises

Exercise 1: Capture HTTP traffic only and save to a file.
Solution:
tshark -i eth0 -f "tcp port 80" -c 1000 -w http_traffic.pcap
Exercise 2: Find all packets containing the word "password" in an HTTP stream.
Solution:
http contains "password"
Exercise 3: Find all TCP SYN packets (connection requests).
Solution:
tcp.flags.syn == 1 and tcp.flags.ack == 0
Exercise 4: Extract all DNS queries from a capture file.
Solution:
tshark -r capture.pcap -Y dns -T fields -e dns.qry.name

🎓 Key Takeaways (Wireshark Mastery)

  • ✅ Capture filters drop packets before capture (use BPF syntax).
  • ✅ Display filters hide packets after capture (more powerful).
  • ✅ Follow TCP/UDP streams to reconstruct entire conversations.
  • ✅ Use logical operators (`and`, `or`, `not`) to combine filters.
  • ✅ Statistics (Endpoints, Conversations, I/O Graph) reveal patterns.
  • ✅ tshark brings Wireshark's power to the command line.
  • ✅ Always capture legally. Only analyze your own traffic or traffic you have permission to inspect.
