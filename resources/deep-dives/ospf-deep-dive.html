🔄 OSPF Deep Dive

Master OSPF internals for enterprise networks. This deep dive covers LSA types (1-7), special area types (Stub, NSSA, Totally Stubby, Totally NSSA), virtual links, route filtering, summarization, and advanced troubleshooting techniques used by CCNP/CCIE professionals.

💡 Prerequisite: Complete the core OSPF lesson first. This deep dive assumes you understand OSPF fundamentals (areas, DR/BDR, cost metric).

📊 LSA Types 1-7 — The Language of OSPF

OSPF routers exchange Link State Advertisements (LSAs) to build the network map. Understanding LSA types is essential for advanced troubleshooting and design.

TypeNameWhat It ContainsFlooded Where?Who Generates?
Type 1 Router LSA Lists all router's interfaces, neighbors, and link types (point-to-point, transit, stub) Within area only Every OSPF router
Type 2 Network LSA Lists all routers attached to a broadcast segment (Ethernet) — DR generates this Within area only Designated Router (DR)
Type 3 Network Summary LSA Advertises networks from one area to another (inter-area routes) Throughout OSPF domain Area Border Router (ABR)
Type 4 ASBR Summary LSA Points to the ASBR (needed to reach external routes) Throughout OSPF domain ABR (from ASBR's area)
Type 5 External LSA Advertises routes redistributed from other protocols (static, RIP, BGP, connected) Throughout OSPF domain (except stub areas) Autonomous System Border Router (ASBR)
Type 6 Group Membership LSA Used for multicast OSPF (MOSPF) — obsolete, not supported on modern Cisco IOS Not used —
Type 7 NSSA External LSA External routes inside a Not-So-Stubby Area (Type 5 alternative for NSSA) Within NSSA only ASBR inside NSSA
💡 Memory Aid: Remember LSA types by function:
• Type 1 & 2 → Within area topology
• Type 3 & 4 → Inter-area connectivity
• Type 5 → External routes (redistributed)
• Type 7 → External routes in NSSA

📡 Special Area Types — Reduce Routing Tables

OSPF areas can be modified to block certain LSAs, reducing routing table size and improving performance.

Area TypeBlocked LSAsDefault Route Injected?Use Case
Standard Area None (Type 3,4,5 allowed) No Regular areas
Stub Area Type 4, Type 5 Yes (Type 3 default) Remote sites without external routes
Totally Stubby (Cisco) Type 3,4,5 (only default) Yes (only default) Maximum reduction for branch offices
NSSA (Not-So-Stubby) Type 4,5 (Type 7 allowed for redistribution) Yes (Type 3 default) Stub area that needs to redistribute external routes
Totally NSSA (Cisco) Type 3,4,5 (Type 7 allowed + default only) Yes (only default) Maximum reduction with redistribution capability

Configuration Examples:

! Stub Area
Router(config-router)# area 1 stub

! Totally Stubby Area (Cisco proprietary — blocks Type 3 as well)
Router(config-router)# area 1 stub no-summary

! NSSA
Router(config-router)# area 1 nssa

! Totally NSSA
Router(config-router)# area 1 nssa no-summary
⚠️ Important: All routers in the same area must agree on the area type. Mixing stub and non-stub configuration breaks neighbor relationships!

🔗 Virtual Links — Connecting Areas Through Area 0

Virtual links allow a non-backbone area to connect to the backbone (Area 0) through another non-backbone area. Use sparingly — indicates poor design!

When to Use Virtual Links:

  • ✅ Merging two OSPF networks after a company acquisition
  • ✅ Temporary solution while redesigning area layout
  • ✅ When physical connectivity prevents direct Area 0 connection

Configuration:

! On Router A (connecting Area 1 to Area 0 via Area 2)
router ospf 1
area 2 virtual-link 2.2.2.2 ! 2.2.2.2 is Router B's router-id

! On Router B (other end of the virtual link)
router ospf 1
area 2 virtual-link 1.1.1.1 ! 1.1.1.1 is Router A's router-id
📋 Verification:
Router# show ip ospf virtual-links
Virtual Link OSPF_VL0 to router 2.2.2.2 is up
Run as demand circuit
Transit area 2, via interface Serial0/0/0, Cost: 64
⚠️ Design Warning: Virtual links complicate troubleshooting and can cause routing loops. Always prefer physical redesign over virtual links.

🗺️ OSPF Route Summarization — Shrink the Routing Table

Inter-Area Summarization (on ABR):

router ospf 1
area 1 range 192.168.0.0 255.255.252.0
! Summarizes all routes from Area 1 into 192.168.0.0/22

External Route Summarization (on ASBR):

router ospf 1
summary-address 10.0.0.0 255.255.0.0
! Summarizes redistributed routes into 10.0.0.0/16
💡 Best Practice: Summarize at area boundaries whenever possible. Each summarized route hides flapping subnets from the rest of the network.

🔐 OSPF Authentication — MD5 vs SHA

MD5 Authentication (Classic, widely supported):

router ospf 1
area 0 authentication message-digest
!
interface serial0/0/0
ip ospf message-digest-key 1 md5 SecurePass123

SHA Authentication (Stronger, IOS 15.2+):

router ospf 1
area 0 authentication message-digest
security credential
key-chain OSPF-KEY
!
key chain OSPF-KEY
key 1
key-string SecurePass123
cryptographic-algorithm hmac-sha-256
💡 Recommendation: Use SHA-256 when all routers support it. MD5 is acceptable for legacy networks but considered weak.

🛠️ Advanced OSPF Troubleshooting

1. Check LSA Flooding Issues:

Router# show ip ospf database
Router# show ip ospf database external ! Type 5 LSAs
Router# show ip ospf database summary ! Type 3 LSAs

2. View SPF Calculations (CPU intensive!):

Router# show ip ospf statistic ! View SPF run history
Router# debug ip ospf spf ! Live SPF debugging (use sparingly)

3. Check OSPF Neighbor Issues:

Router# show ip ospf neighbor detail
Router# show ip ospf interface serial0/0/0
! Check: area ID, hello/dead timers, authentication, MTU

4. Common OSPF Problems & Fixes:

  • Neighbors stuck in EXSTART/EXCHANGE → MTU mismatch. Fix: `ip ospf mtu-ignore` on interface
  • Neighbors stuck in 2-WAY → DR/BDR election issue (check priority)
  • Missing routes → Stub area blocking Type 5 LSAs, or summarization too aggressive
  • High CPU from SPF → Network flapping or too many LSAs (summarize!)

Router(config)# ipv6 unicast-routing

📝 Advanced Practice Questions

Q1: Your branch office router has a 250,000 route routing table. How can you reduce it?
Answer: Configure the area as Totally Stubby (`area X stub no-summary`). This blocks Type 3,4,5 LSAs, injecting only a default route — drastically reducing the table.
Q2: Two routers in the same area can't form adjacency — they're stuck in 2-WAY state. What's the likely cause?
Answer: They are on a broadcast network (Ethernet) with identical DR priority. OSPF doesn't elect DR/BDR when both priorities are 0. Set priority on at least one router: `ip ospf priority 1`.
Q3: You need to redistribute static routes into OSPF from a branch router, but the area is Totally Stubby. What should you do?
Answer: Change the area to NSSA (or Totally NSSA). NSSA allows Type 7 LSAs for redistributed routes while still blocking Type 5 LSAs.
Q4: What LSA type advertises external routes, and which router generates it?
Answer: Type 5 (External LSA), generated by an ASBR (Autonomous System Border Router).
Q5: Your OSPF database has thousands of Type 3 LSAs from multiple areas. How can you reduce them?
Answer: Configure inter-area summarization on the ABR: `area 1 range 192.168.0.0 255.255.252.0` to advertise one summary instead of many specific routes.

🎓 Key Takeaways (OSPF Deep Dive)

  • ✅ LSA Types 1-7 — Know what each does and where it floods
  • ✅ Stub areas — Block Type 4/5 LSAs (reduce routing tables)
  • ✅ Totally Stubby — Also blocks Type 3 (Cisco proprietary, maximum reduction)
  • ✅ NSSA — Allows redistribution into stub areas (Type 7 LSAs)
  • ✅ Virtual links — Last resort for connecting to Area 0 (avoid when possible)
  • ✅ Summarization — Configure on ABRs (`area X range`) or ASBRs (`summary-address`)
  • ✅ Authentication — Use SHA over MD5 when possible
  • ✅ OSPFv3 — No network statements; authentication via IPsec
