⚡ EIGRP Advanced Deep Dive

Beyond the basics — master EIGRP for enterprise networks. This deep dive covers stub routing, route summarization, wide metrics, named mode EIGRP, and advanced troubleshooting techniques used by CCNP/CCIE professionals.

💡 Prerequisite: Complete the core EIGRP lesson first. This deep dive assumes you understand EIGRP fundamentals (AS numbers, successors, feasible successors, variance).

📡 EIGRP Stub Routing — Control Query Scope

Stub routing is critical for large networks. It tells a router: "I am a stub — don't query me for alternate paths." This prevents unnecessary query floods when a route goes down.

Why Stub Routing Matters:

  • ✅ Reduces query scope — Stub routers don't propagate queries
  • ✅ Faster convergence — No waiting for stub routers to respond
  • ✅ Prevents SIA (Stuck-In-Active) errors — Major cause of EIGRP instability
  • ✅ Required for hub-and-spoke WAN designs — Branch routers should always be stubs

Configuration:

Router(config)# router eigrp 100
Router(config-router)# eigrp stub
! Default: connected + summary routes advertised

! More restrictive options:
Router(config-router)# eigrp stub receive-only ! Does not advertise any routes
Router(config-router)# eigrp stub connected ! Advertises only connected routes
Router(config-router)# eigrp stub static ! Advertises static routes
Router(config-router)# eigrp stub redistributed ! Advertises redistributed routes
Router(config-router)# eigrp stub leak-map MYMAP ! Selectively leak specific routes
📋 Verification:
Router# show ip eigrp neighbors detail
! Look for "Stub Peer Advertising" section
Router# show ip protocols | section eigrp stub
! Shows stub configuration
💡 Best Practice: Every branch router should be configured as `eigrp stub connected summary`. This prevents query floods from destabilizing the WAN.

🗺️ EIGRP Route Summarization — Shrink the Routing Table

Route summarization reduces routing table size, improves stability, and hides flapping subnets.

Manual Summarization (on interface):

Router(config)# interface serial0/0/0
Router(config-if)# ip summary-address eigrp 100 192.168.0.0 255.255.252.0
! Summarizes 192.168.0.0/22 (covers 192.168.0.0-192.168.3.0)

Benefits of Summarization:

  • ✅ Smaller routing tables — One route instead of 4+
  • ✅ Query boundary — Summaries block queries from propagating
  • ✅ Flapping suppression — A flapping subnet doesn't trigger updates across the summary
⚠️ Warning: Over-summarization can cause routing loops or black holes. Always ensure your summary includes all subnets reachable via that interface!

Verifying Summarization:

Router# show ip route | include "192.168.0.0/22"
D 192.168.0.0/22 [90/409600] via 10.0.0.2, 00:01:23, Serial0/0/0

📏 EIGRP Wide Metrics — Beyond 10 Gbps

Classic EIGRP metrics max out at 10 Gbps. **Wide metrics** support higher bandwidth interfaces (40G, 100G) and use 64-bit calculations.

FeatureClassic EIGRPEIGRP Wide Metrics
Metric width 32-bit 64-bit
Max bandwidth support 10 Gbps 4.2 Tbps (theoretically unlimited)
Router compatibility All IOS routers IOS 15.2+ / IOS-XE only

Enabling Wide Metrics:

Router(config)# router eigrp 100
Router(config-router)# metric version 64bit ! Enables wide metrics
! Also enables classic metrics for backward compatibility
Router(config-router)# metric version 64bit 1-65535 ! Specify AS range
💡 Design Note: Wide metrics are only supported on routers running IOS 15.2+ or IOS-XE. Mixed environments require careful planning — wide metrics routers can still peer with classic routers using scaled metrics.

📁 Named Mode EIGRP — Modern Configuration

Named mode EIGRP is the **modern configuration method** (IOS 15+). It organizes settings into **address families** (IPv4, IPv6) and **topologies** (base, multicast).

Traditional vs Named Mode:

! TRADITIONAL (still works, but limited)
router eigrp 100
network 192.168.1.0
variance 2

! NAMED MODE (recommended for new deployments)
router eigrp MY-COMPANY
address-family ipv4 autonomous-system 100
network 192.168.1.0
variance 2
topology base
summary-address 192.168.0.0 255.255.252.0
exit-af-topology
exit-address-family

Named Mode Benefits:

  • ✅ IPv4 and IPv6 unified config — Same router process handles both
  • ✅ Multi-topology routing — Different paths for voice vs data
  • ✅ Simpler migration — Change settings without restarting the protocol
  • ✅ Per-address-family settings — Different variance for IPv4 vs IPv6

Multi-Topology Routing (Voice vs Data):

router eigrp MY-COMPANY
address-family ipv4 autonomous-system 100
topology base
! Data traffic uses standard metrics
exit-af-topology
topology voice
! Voice traffic prefers low-latency links
metric rib-metric 100 1000 255 1 1500
exit-af-topology
exit-address-family
⚠️ Compatibility Note: Named mode EIGRP can peer with classic mode EIGRP (both speak the same protocol). However, named mode features (wide metrics, multi-topology) require named mode peers.

🔐 EIGRP Authentication — Beyond MD5

Classic EIGRP supports MD5 authentication. Named mode adds **SHA (Secure Hash Algorithm)** for stronger security.

MD5 Authentication (Classic):

key chain EIGRP-KEY
key 1
key-string SecurePass123
!
interface serial0/0/0
ip authentication mode eigrp 100 md5
ip authentication key-chain eigrp 100 EIGRP-KEY

SHA Authentication (Named Mode):

router eigrp MY-COMPANY
address-family ipv4 autonomous-system 100
authentication mode hmac-sha-256 authentication key-chain EIGRP-KEY
exit-address-family
💡 Recommendation: Use SHA-256 when all routers support it. MD5 is acceptable for mixed environments but considered legacy.

🛠️ Troubleshooting SIA (Stuck-In-Active) — The #1 EIGRP Problem

SIA occurs when a router waits too long for replies to queries. This often indicates a problem with **stub configuration** or **query scope**.

Common SIA Causes:

  • ❌ Branch routers not configured as stubs
  • ❌ High-latency links (satellite, cellular)
  • ❌ Large routing tables with many queries
  • ❌ Router CPU overload (slow to respond)

Diagnostic Commands:

! Show active routes (routes in ACTIVE state — problematic)
Router# show ip eigrp topology active

! Show stuck routes
Router# show ip eigrp topology pending

! Increase active timer (default 3 minutes)
Router(config-router)# timers active-time 5

! Disable SIA checking (not recommended for production)
Router(config-router)# no eigrp log-neighbor-warnings
⚠️ SIA Prevention: Always configure `eigrp stub` on branch routers. This is the single most effective SIA mitigation.

📊 Advanced Verification Commands (CCNP/CCIE Level)

🔍 EIGRP Topology Table Details:
Router# show ip eigrp topology 192.168.1.0/24
! Shows detailed information about a specific route
🔍 EIGRP Traffic Statistics:
Router# show ip eigrp traffic
! Shows queries, replies, updates — helps diagnose SIA
🔍 EIGRP Interface Details:
Router# show ip eigrp interfaces detail
! Shows hello interval, hold time, split horizon, summary addresses
🔍 EIGRP Events (Debugging):
Router# debug eigrp fsm
! Shows Finite State Machine transitions (when routes go active/passive)
Router# debug eigrp packets query
! Shows only query packets — less noisy than full debug

🌐 EIGRP for IPv6 — Configuration Differences

EIGRP for IPv6 uses a **similar configuration** but with some key differences.

! Enable IPv6 routing (required)
Router(config)# ipv6 unicast-routing

! Named mode EIGRP for IPv6
Router(config)# router eigrp MY-COMPANY
Router(config-router)# address-family ipv6 autonomous-system 100
Router(config-router-af)# network 2001:db8::/64
Router(config-router-af)# exit-address-family

! Interface configuration
Router(config)# interface gig0/0
Router(config-if)# ipv6 eigrp 100
💡 Note: EIGRP for IPv6 requires explicit interface configuration (`ipv6 eigrp AS`). It does NOT use network statements like IPv4.

📝 Advanced Practice Questions

Q1: Your branch router is causing SIA across the WAN. What configuration is likely missing?
Answer: `eigrp stub` on the branch router. Without stub configuration, EIGRP queries propagate to all routers, causing delays.
Q2: Your network has 100Gbps links. Classic EIGRP can't differentiate between 10G and 100G. How do you fix this?
Answer: Enable wide metrics with `metric version 64bit` in router configuration mode. This supports up to 4.2 Tbps.
Q3: What's the advantage of named mode EIGRP over traditional configuration?
Answer: Named mode supports wide metrics, multi-topology routing (voice vs data), unified IPv4/IPv6 configuration, and SHA authentication.
Q4: You need to advertise 5 branch subnets to headquarters. Should you use separate route advertisements or summarization?
Answer: Summarization! Configure `ip summary-address eigrp 100 192.168.0.0 255.255.248.0` to advertise one route covering all 5 subnets.
Q5: What is the difference between classic MD5 and SHA authentication for EIGRP?
Answer: MD5 is 128-bit, considered legacy but widely supported. SHA-256 is 256-bit, stronger, and recommended for modern networks (requires named mode).

🎓 Key Takeaways (EIGRP Advanced)

  • ✅ Stub routing — Critical for preventing SIA. Configure `eigrp stub` on all branch routers
  • ✅ Summarization — Reduces routing table size, hides flapping routes, blocks queries
  • ✅ Wide metrics — Required for 10G+ links. Enable with `metric version 64bit`
  • ✅ Named mode — Modern configuration (IOS 15+), supports multi-topology routing
  • ✅ SHA authentication — Use instead of MD5 when all routers support it
  • ✅ SIA troubleshooting — Check stub configuration first, then active timers
  • ✅ IPv6 EIGRP — Requires explicit interface configuration, not network statements
