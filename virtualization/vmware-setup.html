⚡ VMware Setup — Complete Guide

VMware is the industry leader in virtualization — from free desktop players to enterprise data center hypervisors. This guide focuses on VMware Workstation Pro/Player for personal labs and security testing.

💡 Real-world: Most corporate labs, CTF competitions, and malware analysis environments run on VMware.

⬇️ Step 1: Download & Install VMware (Linux First)

Download VMware Workstation Player for Linux

# Download from official site:
https://www.vmware.com/products/workstation-player.html

# Direct download (Linux 64-bit .bundle file)
wget https://download3.vmware.com/software/PLAYER-2024/VMware-Player-Full-17.5.2-23775571.x86_64.bundle

Installation (Ubuntu/Debian - Linux)

# Make the bundle executable
chmod +x VMware-Player-*.bundle

# Install dependencies (required for kernel modules)
sudo apt update
sudo apt install build-essential gcc make linux-headers-$(uname -r) -y

# Run the installer (as root)
sudo ./VMware-Player-*.bundle

# Follow GUI prompts → Accept license → Finish

Alternative: Install via Ubuntu Repository (Older Version)

sudo apt install vmware-workstation-player -y

Installation (Fedora/RHEL - Linux)

# Install dependencies
sudo dnf install kernel-devel kernel-headers gcc make perl -y

# Run the installer
sudo ./VMware-Player-*.bundle

Post-Installation: Build Kernel Modules

# After first launch, VMware will ask to compile kernel modules
# If it fails, manually run:
sudo vmware-modconfig --console --install-all

# If you get "Unable to install all modules", try:
sudo apt install linux-headers-$(uname -r) build-essential -y
sudo reboot
💡 Linux tip: After kernel updates, you may need to recompile VMware modules: sudo vmware-modconfig --console --install-all

🖥️ Step 2: Create Your First Virtual Machine (Ubuntu Server)

Download Ubuntu Server ISO

https://ubuntu.com/download/server

Create New VM (GUI Method)

1. Open VMware Workstation Player → "Create a New Virtual Machine"
2. Select "Installer disc image file (iso)" → Browse to Ubuntu ISO
3. Guest operating system: Linux → Ubuntu 64-bit
4. Name: "Ubuntu-Lab-Server" → Location: ~/vmware/
5. Disk size: 25GB → "Store virtual disk as single file"
6. Customize Hardware →
• Memory: 2048 MB
• Processors: 2 cores
• Network: NAT (for internet)
7. Click "Finish"

Create VM via Command Line (Linux Power Users)

# Create VM directory
mkdir -p ~/vmware/Ubuntu-Lab-Server

# Create VMX configuration file
nano ~/vmware/Ubuntu-Lab-Server/Ubuntu-Lab-Server.vmx

# Add content:
.encoding = "UTF-8"
config.version = "8"
virtualHW.version = "19"
displayName = "Ubuntu-Lab-Server"
guestOS = "ubuntu-64"
memSize = "2048"
numvcpus = "2"

# Create virtual disk
vmware-vdiskmanager -c -s 25GB -a lsilogic -t 0 ~/vmware/Ubuntu-Lab-Server/Ubuntu-Lab-Server.vmdk

# Register VM with VMware
vmware -x ~/vmware/Ubuntu-Lab-Server/Ubuntu-Lab-Server.vmx
💡 CLI power: Automate VM creation with vmrun, vmware-vdiskmanager, and vmware-cmd!

🌐 Step 3: Understanding VMware Network Modes

🟢 NAT (Default)
VM shares host's IP. Host → VM not possible unless port forwarding. Best for: Internet access, downloading packages.
🔵 Bridged
VM appears as separate device on physical network. Gets own IP from router. Best for: Running services, testing network tools.
🟡 Host-Only
VM can talk to host and other Host-Only VMs. No DHCP (manual IP config). Best for: Isolated labs, malware analysis.
⚪ Custom (VMnet)
Configure virtual networks (VMnet0-19). Advanced: Multiple isolated networks, DHCP servers. Best for: Complex network simulations.
⚠️ Security tip: For malware analysis, use Host-Only mode. Disconnect VM from the internet completely!

Configuring Networks via Command Line (Linux)

# List all running VMs
vmrun list

# Change network mode (requires VM to be powered off)
vmware-cmd /path/to/VM.vmx setnetworkadapter virtulen name "HostOnly"

# Edit VMX file directly to change network type
echo "ethernet0.connectionType = 'bridged'" >> /path/to/VM.vmx
echo "ethernet0.networkName = 'Bridged'" >> /path/to/VM.vmx

📸 Step 5: Snapshots (Workstation Pro Only — Your Undo Button)

# GUI Method (Workstation Pro):
VM → Snapshot → Take Snapshot → Name it "Clean install"

# Restore: VM → Snapshot → Snapshot Manager → Go To

# CLI Method (vmrun):
vmrun snapshot /path/to/VM.vmx "Clean install"
vmrun listSnapshots /path/to/VM.vmx
vmrun revertToSnapshot /path/to/VM.vmx "Clean install"
⚠️ Player limitation: VMware Workstation Player does NOT support snapshots. Use Workstation Pro or VirtualBox if snapshots are essential.

📋 Step 6: Cloning VMs (Pro Only — Rapid Lab Building)

# GUI Method (Workstation Pro):
Right-click VM → Manage → Clone → Follow wizard

# CLI Method (vmrun - Linux):
vmrun clone /path/to/source.vmx /path/to/clone.vmx full

# Example: Clone Ubuntu-Server to Ubuntu-Client
vmrun clone ~/vmware/Ubuntu-Server/Ubuntu-Server.vmx ~/vmware/Ubuntu-Client/Ubuntu-Client.vmx full
💡 Clone types: Full clone (independent copy) vs Linked clone (shares disk, saves space).

🔧 Common VMware Issues & Fixes

❌ "Unable to build kernel module" on Linux
Fix:
sudo apt install build-essential linux-headers-$(uname -r)
sudo vmware-modconfig --console --install-all
sudo reboot
❌ VMware won't start after kernel update
Fix: Recompile modules:
sudo vmware-modconfig --console --install-all
❌ VM has no internet (NAT mode)
Fix: Check Virtual Network Editor: Edit → Virtual Network Editor → Restore Defaults. Then restart VM.
❌ "This kernel requires an X86-64 CPU" error
Fix: Enable virtualization in BIOS: Reboot → Enter BIOS → Enable Intel VT-x or AMD SVM.

📝 Practice Questions

Q1: What is the main difference between VMware Workstation Player and Pro?

Answer: Player is free but lacks snapshots, cloning, teams, and encryption. Pro adds these enterprise features.

Q2: Which network mode should you use for malware analysis (no internet access)?

Answer: Host-Only mode — isolates VMs from the physical network and internet.

Q3: What command rebuilds VMware kernel modules on Linux after a kernel update?

Answer: sudo vmware-modconfig --console --install-all

Q4: Why would you choose VMware over VirtualBox?

Answer: Better performance (3D acceleration), enterprise features (encryption, teams), and industry standard for corporate labs.
