🛡️ Advanced ACLs — Access Control Lists

Access Control Lists (ACLs) are packet filters that permit or deny traffic based on source/destination IP, port, protocol, and more. Every router and firewall uses ACLs for traffic control and security.

💡 Prerequisite: Basic IP addressing and subnetting. ACLs are essential for network security and traffic management.

📊 Types of ACLs

TypeNumber RangeFilters Based OnUse Case
Standard ACL 1-99 or 1300-1999 Source IP address only Blocking entire subnets (simple)
Extended ACL 100-199 or 2000-2699 Source/destination IP, protocol (TCP/UDP), port, flags Granular traffic control (most common)

📝 Standard ACL Configuration

! Create standard ACL (permit/deny source IP)
Router(config)# access-list 10 permit 192.168.1.0 0.0.0.255
Router(config)# access-list 10 deny any

! Apply to interface (direction matters!)
Router(config)# interface gigabitEthernet 0/0
Router(config-if)# ip access-group 10 in

! Verification
Router# show access-lists
Router# show ip interface gig0/0
📋 Example: Block all traffic from a specific host
Router(config)# access-list 5 deny host 10.0.0.100
Router(config)# access-list 5 permit any
Router(config)# interface gig0/0
Router(config-if)# ip access-group 5 in

🔧 Extended ACL Configuration (Most Common)

! Extended ACL syntax:
! access-list NUMBER permit|deny PROTOCOL SOURCE_IP WILDCARD DEST_IP WILDCARD [eq PORT]

! Allow web traffic from internal network to internet
Router(config)# access-list 100 permit tcp 192.168.1.0 0.0.0.255 any eq 80
Router(config)# access-list 100 permit tcp 192.168.1.0 0.0.0.255 any eq 443

! Allow established connections (return traffic)
Router(config)# access-list 100 permit tcp any any established

! Allow DNS
Router(config)# access-list 100 permit udp any any eq 53

! Deny everything else (implicit deny at end)
Router(config)# interface gigabitEthernet 0/0
Router(config-if)# ip access-group 100 out
💡 Key Insight: Extended ACLs process from top to bottom. Place more specific rules first, general rules later. There's an implicit "deny all" at the end.

📛 Named ACLs (More Readable)

Named ACLs use descriptive names instead of numbers — much easier to manage.

! Create named extended ACL
Router(config)# ip access-list extended BLOCK-BAD-TRAFFIC
Router(config-ext-nacl)# deny tcp any any eq 4444
Router(config-ext-nacl)# deny udp any any range 1337 1339
Router(config-ext-nacl)# permit ip any any
Router(config-ext-nacl)# exit

! Apply to interface
Router(config)# interface gigabitEthernet 0/0
Router(config-if)# ip access-group BLOCK-BAD-TRAFFIC in

⚡ Advanced ACL Features

1. Established Flag (Return Traffic)

! Allow return TCP traffic (if ACK or RST flag is set)
Router(config)# access-list 100 permit tcp any any established

2. TCP Flags (Special Control)

! Block TCP SYN scans (only SYN flag, no ACK)
Router(config)# access-list 100 deny tcp any any syn

! Block TCP reset attacks
Router(config)# access-list 100 deny tcp any any rst

3. Port Ranges

! Block range of ports
Router(config)# access-list 100 deny tcp any any range 6000 6010

! Allow multiple specific ports
Router(config)# access-list 100 permit tcp any any eq 22
Router(config)# access-list 100 permit tcp any any eq 80
Router(config)# access-list 100 permit tcp any any eq 443

4. ICMP Control

! Allow ping (echo request) but block other ICMP
Router(config)# access-list 100 permit icmp any any echo
Router(config)# access-list 100 permit icmp any any echo-reply
Router(config)# access-list 100 deny icmp any any

5. Established Connections (Reflexive ACLs)

! Allow inside users to browse web, block outside from initiating
Router(config)# ip access-list extended OUTBOUND
Router(config-ext-nacl)# permit tcp 192.168.1.0 0.0.0.255 any reflect INSIDE-TRAFFIC

Router(config)# ip access-list extended INBOUND
Router(config-ext-nacl)# evaluate INSIDE-TRAFFIC
Router(config-ext-nacl)# deny ip any any

⏰ Time-Based ACLs (Restrict by Schedule)

! Define time range
Router(config)# time-range WORK_HOURS
Router(config-time-range)# periodic weekdays 8:00 to 17:00
Router(config-time-range)# exit

! Block social media during work hours
Router(config)# access-list 100 deny tcp any any eq 80 time-range WORK_HOURS
Router(config)# access-list 100 deny tcp any any eq 443 time-range WORK_HOURS

✅ ACL Best Practices

  • ✅ Place extended ACLs as close to source as possible — Saves bandwidth
  • ✅ Standard ACLs go as close to destination — They can't see destination IPs
  • ✅ Order matters — Most specific rules first, general at bottom
  • ✅ Use named ACLs — Easier to understand than numbers
  • ✅ Add remarks — Document what each rule does
  • ✅ Test with permit any any temporarily — Avoid locking yourself out
  • ✅ Never edit ACLs remotely without fallback — You can lose access
! Add remarks to ACLs (documentation)
Router(config)# access-list 100 remark Allow web traffic from internal network
Router(config)# access-list 100 permit tcp 192.168.1.0 0.0.0.255 any eq 80
Router(config)# access-list 100 remark Allow HTTPS
Router(config)# access-list 100 permit tcp 192.168.1.0 0.0.0.255 any eq 443

🔧 Troubleshooting ACLs

Common Issues:

  • Traffic blocked unexpectedly — Use show access-lists to see hit counters
  • Order wrong — Rules processed top-down; more specific must come first
  • Implicit deny at end — If nothing matches, traffic is dropped
  • Wrong direction — Check if ACL applied to correct interface and direction (in/out)
! View ACL hit counts (increments when rule matches)
Router# show access-lists 100

! Clear ACL counters
Router# clear access-list counters

! Debug ACL (use carefully - CPU intensive)
Router# debug ip packet 100 detail

🔐 ACLs on VTY Lines (SSH/Telnet Access Control)

Control who can remotely access the router itself.

! Allow only specific subnet to SSH into router
Router(config)# access-list 50 permit 192.168.10.0 0.0.0.255
Router(config)# access-list 50 deny any

! Apply to VTY lines
Router(config)# line vty 0 4
Router(config-line)# access-class 50 in
Router(config-line)# transport input ssh

🌐 Real-World Security Scenarios

🔴 Scenario 1: Block SSH brute-force attempts
access-list 100 deny tcp any any eq 22
access-list 100 permit ip any any
→ Place this ACL on the outside interface to block all SSH from internet.
🔴 Scenario 2: Allow only specific webserver access
access-list 100 permit tcp host 10.0.0.50 any eq 80
access-list 100 permit tcp host 10.0.0.50 any eq 443
access-list 100 deny ip any any
→ Only webserver 10.0.0.50 can access internet.
🔴 Scenario 3: Block port scanning
access-list 100 deny tcp any any fragments
→ Drops fragmented packets (common in nmap scans)

📝 Practice Questions

Q1: What is the difference between a standard and extended ACL?
Answer: Standard ACLs filter only on source IP address. Extended ACLs filter on source/destination IP, protocol, port, and TCP flags.
Q2: What is the "established" keyword used for in ACLs?
Answer: It matches TCP packets with the ACK or RST flag set — allowing return traffic for connections initiated from inside while blocking outside-initiated connections.
Q3: What happens at the end of every ACL?
Answer: There is an implicit "deny all" (deny ip any any). If no rules match, traffic is dropped.
Q4: What is the proper placement of extended ACLs?
Answer: Extended ACLs should be placed as close to the source of traffic as possible to save bandwidth.
Q5: How do you restrict SSH access to a router from only one subnet?
Answer: Create a standard ACL and apply it to VTY lines using access-class.

🎓 Key Takeaways (Advanced ACLs)

  • ✅ Standard ACL (1-99) — Source IP only. Place near destination
  • ✅ Extended ACL (100-199) — Source/destination IP, protocol, port. Place near source
  • ✅ Named ACL — Descriptive names, easier to manage
  • ✅ Order matters — Specific rules first, general last. Implicit deny at end
  • ✅ Established flag — Allows return TCP traffic (ACK/RST set)
  • ✅ Time-based ACLs — Block/allow during specific hours
  • ✅ Always test ACLs — Temporary permit all before applying remotely
  • ✅ VTY access-class — Control who can SSH/telnet to router
