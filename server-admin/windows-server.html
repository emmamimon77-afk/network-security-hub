🪟 Windows Server Security — Complete Guide

Windows Server runs 70%+ of enterprise servers worldwide. Mastering its security tools is essential for any network professional.

🎯 Why Windows Server Security Matters

  • ✅ Dominant in enterprises — Active Directory, Exchange, file servers
  • ✅ Frequent attack target — Ransomware, credential theft, lateral movement
  • ✅ Powerful built-in tools — Event Viewer, PowerShell, Windows Defender
  • ✅ Compliance requirements — PCI-DSS, HIPAA, SOX demand Windows security
💡 Real-world: Most ransomware attacks target Windows Server because it holds the crown jewels (Active Directory, file shares).

🛡️ Core Windows Server Security Tools

⚡ PowerShell for Security (The Modern Way)

User & Group Management

Get-LocalUser # List all local users
Get-LocalGroup # List all local groups
Get-LocalGroupMember -Name "Administrators" # Who is admin?
New-LocalUser -Name "secadmin" -Password (Read-Host -AsSecureString)
Add-LocalGroupMember -Group "Administrators" -Member "secadmin"

Network & Firewall

Get-NetFirewallRule | Where-Object {$_.Enabled -eq "True"} # List active rules
New-NetFirewallRule -DisplayName "Allow SSH" -Direction Inbound -Protocol TCP -LocalPort 22 -Action Allow
Get-NetTCPConnection -State Listen # What's listening?

Logs & Auditing

Get-EventLog -LogName Security -EntryType Error -Newest 20 # Last 20 security errors
Get-WinEvent -FilterHashTable @{LogName='Security'; ID=4625} -MaxEvents 10 # Failed logins
Get-Process | Sort-Object CPU -Descending | Select-Object -First 10 # Top 10 CPU processes
💡 PowerShell is essential — most security tasks are faster in PowerShell than in the GUI.

📜 Event Log Analysis (Critical Event IDs)

Event IDWhat It MeansWhy It Matters
4624 Successful logon Normal, but spikes at odd hours = suspicious
4625 Failed logon Brute-force indicator — too many = attack
4648 Logon using explicit credentials Possible lateral movement (pass-the-hash)
4672 Special privileges assigned to new logon Admin logon — monitor carefully
4688 New process created Detect malware execution (with command-line logging enabled)
7045 New service installed Persistence mechanism — malware loves this
# Monitor failed logons in real-time
Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4625} -MaxEvents 50 | Format-List TimeCreated, Message

🛡️ Security Hardening Checklist (Critical First Steps)

✅ Account & Password Policies
  • Disable default Administrator account (rename it)
  • Enforce complex passwords (12+ chars, complexity enabled)
  • Set account lockout policy (5 attempts, lock 15 min)
  • Remove unnecessary local users and groups
✅ Network & Firewall
  • Enable Windows Defender Firewall (block inbound by default)
  • Disable insecure protocols: SMBv1, LLMNR, NetBIOS
  • Use RDP with Network Level Authentication (NLA)
  • Change RDP default port (3389 → custom)
✅ Logging & Monitoring
  • Enable PowerShell logging (script block + module)
  • Enable command-line auditing (see exactly what was run)
  • Forward logs to SIEM (Splunk, ELK, Sentinel)
  • Set log size limits (prevent overwrite)
✅ Application Control
  • Enable Windows Defender Antivirus (real-time protection)
  • Consider AppLocker or Windows Defender Application Control (WDAC)
  • Disable macros in Office (GPO)
  • Remove unnecessary software

🏢 Active Directory Security (The Heart of Windows Enterprise)

💡 Active Directory (AD) is the most attacked component in Windows networks. Compromise AD = compromise the entire organization.

Critical AD Security Commands

# List all domain admins
Get-ADGroupMember -Identity "Domain Admins"

# Find inactive user accounts (security risk!)
Search-ADAccount -AccountInactive -TimeSpan 90.00:00:00

# Find accounts with passwords never expiring
Get-ADUser -Filter {PasswordNeverExpires -eq $true}

Common AD Attacks (What to Watch For)

  • Kerberoasting: Extract service account hashes — monitor for event ID 4769 with unusual tickets
  • Golden Ticket: Forged Kerberos tickets — protect KRBTGT account, monitor abnormal TGS requests
  • Pass-the-Hash: Use NTLM hash without password — enable Kerberos only where possible
  • DCShadow: Malicious AD replication — monitor for 4928, 4929 events

🚨 Incident Response Checklist (Windows Server)

# 1. Check active user sessions
quser
net session

# 2. List running processes
Get-Process | Sort-Object CPU -Descending | Select -First 10
tasklist /v

# 3. Check network connections
netstat -ano
Get-NetTCPConnection -State Established

# 4. Check scheduled tasks (persistence)
schtasks /query /fo LIST /v
Get-ScheduledTask | Where-Object {$_.State -ne "Disabled"}

# 5. Check services
Get-Service | Where-Object {$_.Status -eq "Running"}

# 6. Check recent logins
Get-EventLog -LogName Security -InstanceId 4624 -Newest 20 | Format-List TimeGenerated, Message

📝 Practice Questions

Q1: What PowerShell command lists all local users on a Windows Server?

Answer: Get-LocalUser

Q2: Which Event ID indicates a failed login attempt?

Answer: Event ID 4625 (failed logon).

Q3: What tool would you use to see all programs that run at Windows startup?

Answer: Autoruns.exe from Sysinternals Suite.

Q4: How do you check the members of the local Administrators group via PowerShell?

Answer: Get-LocalGroupMember -Name "Administrators"

Q5: What is a Golden Ticket attack in Active Directory?

Answer: A forged Kerberos ticket (TGT) that gives an attacker domain admin rights. Protect the KRBTGT account and rotate its password.

🧪 Lab Exercise: Windows Server Security Investigation

Scenario:

A Windows Server is suspected of compromise. Investigate using built-in tools.

Step 1: Examine active user sessions

quser
net session

Step 2: Check for unusual processes

Get-Process | Sort-Object CPU -Descending | Select -First 10
tasklist /v /fi "STATUS eq RUNNING"

Step 3: Review security events (failed logins)

Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4625} -MaxEvents 20 | Format-List TimeCreated, Message

Step 4: Check persistent mechanisms

Get-ScheduledTask | Where-Object {$_.State -ne "Disabled"}
Get-Service | Where-Object {$_.StartType -eq "Automatic" -and $_.Status -eq "Running"}
💡 Challenge: Use Event Viewer (eventvwr.msc) to create a custom view showing all failed logins (Event ID 4625) from the last 24 hours.

🎓 Key Takeaways (Windows Server Security)

  • ✅ PowerShell is essential — learn `Get-*` commands for investigation
  • ✅ Event IDs are critical — 4624 (logon), 4625 (failed), 4688 (process), 7045 (service)
  • ✅ Enable command-line auditing — see exactly what attackers ran
  • ✅ Rename default Administrator — basic but effective
  • ✅ Sysinternals tools are lifesavers — ProcMon, Autoruns, TCPView
  • ✅ Active Directory security is its own specialty — protect KRBTGT!
