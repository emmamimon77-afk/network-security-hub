🐧 Linux Commands for Security

Essential Linux commands every security professional must know. From monitoring network connections to analyzing logs and hardening systems.

🎯 Why Linux Commands Matter for Security

  • ✅ 80% of cloud servers run Linux — AWS, Azure, GCP
  • ✅ Security tools are Linux-first — Kali, Wireshark, Metasploit, Nmap
  • ✅ Incident response happens in the terminal — GUI is too slow
  • ✅ CTF competitions & certifications — Linux is required (CEH, Security+, OSCP)
💡 Real-world: When a server is compromised, you have minutes to respond. The CLI is your only option.

🌐 Network Investigation (Find Active Connections)

ss (Socket Statistics) — Modern replacement for netstat

ss -tulpn # Show all listening ports with process names
ss -tan # Show all TCP connections (no names)
ss -s # Summary statistics
ss -t state established 'dport = 443' # Filter HTTPS connections
Example output of `ss -tulpn`:
Netid State Recv-Q Send-Q Local Address:Port Peer Address:Port Process
tcp LISTEN 0 128 0.0.0.0:22 0.0.0.0:* users:(("sshd",pid=1234))
tcp LISTEN 0 128 0.0.0.0:443 0.0.0.0:* users:(("nginx",pid=5678))

lsof (List Open Files)

lsof -i :80 # Which process is using port 80?
lsof -i tcp # All TCP connections
lsof -p 1234 # Files opened by process 1234
lsof -u root # Files opened by root user

netstat (Legacy, still works on many systems)

netstat -tulpn # Listening ports (use ss instead when available)
netstat -rn # Routing table
netstat -i # Interface statistics

🔍 Process Monitoring (Find Malicious Processes)

ps (Process Snapshot)

ps aux | grep nginx # Find nginx processes
ps -ef --forest # Tree view (see parent-child)
ps aux --sort=-%cpu | head -10 # Top 10 CPU consumers

top / htop (Real-time process monitoring)

top # Real-time process viewer
htop # Nicer version (install if missing)
# Inside top: 'P' = sort by CPU, 'M' = sort by memory, 'q' = quit
💡 Security use case: A sudden unknown process using 100% CPU could be a cryptominer. Use `kill -9 [PID]` to stop it.

📜 Log Analysis (Where to Look When Something Breaks)

System Logs (Most important)

journalctl -xe # Full system log (systemd)
journalctl -u sshd --since today # SSH logs from today only
tail -f /var/log/syslog # Follow syslog in real-time (Debian/Ubuntu)
tail -f /var/log/messages # Follow syslog (RHEL/CentOS)
dmesg | tail -20 # Kernel log (hardware issues)

Authentication Logs (Failed logins, sudo)

cat /var/log/auth.log | grep Failed # Failed login attempts (Debian)
cat /var/log/secure | grep 'sshd.*Failed' # Failed SSH (RHEL)
sudo cat /var/log/sudo.log # Sudo commands history
last -10 # Last 10 logins
lastb -10 # Last 10 failed logins

Web Server Logs (If running Apache/nginx)

tail -f /var/log/nginx/access.log # Live web traffic
cat /var/log/nginx/error.log # Web server errors
awk '{print $1}' /var/log/nginx/access.log | sort | uniq -c | sort -nr | head -10 # Top 10 IPs
Quick security scan:
cat /var/log/auth.log | grep "Failed password" | awk '{print $9}' | sort | uniq -c | sort -nr
This shows which IPs are brute-forcing SSH!

🛡️ Firewall Commands (View & Manage Rules)

iptables (Legacy, still everywhere)

iptables -L -v -n # List all rules (verbose, numeric)
iptables -L INPUT # List INPUT chain only
iptables -S # Show rules as commands

ufw (Uncomplicated Firewall) — Ubuntu/Debian

ufw status verbose # Check if firewall is active
ufw allow 22/tcp # Allow SSH
ufw deny 80/tcp # Block HTTP
ufw app list # See application profiles

firewalld — RHEL/CentOS/Fedora

firewall-cmd --state # Is it running?
firewall-cmd --list-all # Current configuration
firewall-cmd --add-service=http --permanent # Add HTTP
firewall-cmd --reload # Apply changes

🔐 File Integrity & Permissions (Detect Tampering)

Check for SUID/SGID binaries (privilege escalation risk)

find / -perm -4000 -type f 2>/dev/null # SUID files
find / -perm -2000 -type f 2>/dev/null # SGID files

Find world-writable files (security risk)

find / -perm -0002 -type f 2>/dev/null # World-writable files

Check file integrity with sha256sum

sha256sum important_config.conf # Get hash
# Compare against known good hash to detect changes
💡 Security use case: Run `find / -perm -4000` regularly. Unexpected SUID files are a common backdoor method.

🛡️ System Hardening (Quick Wins)

# Disable root SSH login
sudo sed -i 's/PermitRootLogin yes/PermitRootLogin no/' /etc/ssh/sshd_config
sudo systemctl restart sshd

# Change default SSH port (security through obscurity)
sudo sed -i 's/#Port 22/Port 2222/' /etc/ssh/sshd_config

# Install and configure fail2ban (blocks brute force)
sudo apt install fail2ban -y
sudo systemctl enable fail2ban --now

# Check for unnecessary services
systemctl list-units --type=service --state=running | grep -v systemd

📦 Package Management (Keep Software Secure)

Debian/Ubuntu (apt)

apt list --upgradable # Check for security updates
apt update && apt upgrade -y # Apply all updates
apt autoremove # Remove unused packages

RHEL/CentOS (yum/dnf)

yum check-update # Check for updates
yum update -y # Apply updates
yum list installed | grep -i vulnerable # Check known vulnerabilities
⚠️ Critical: Always keep packages updated. Most breaches exploit unpatched software.

🚨 Real-World Incident Response Checklist

# 1. Check who is logged in
who -a
w

# 2. Check last logins
last -20
lastb -10 # failed logins

# 3. Check running processes
ps aux --sort=-%cpu | head -10
ss -tulpn

# 4. Check for unusual network connections
lsof -i -n | grep ESTABLISHED

# 5. Check recent log entries
journalctl --since "1 hour ago"

# 6. Check for cron jobs (persistence)
crontab -l
cat /etc/crontab

# 7. Check for modified system binaries
find /bin /usr/bin -type f -mtime -1 2>/dev/null

📝 Practice Questions

Q1: What command shows all listening ports with process names?

Answer: ss -tulpn or netstat -tulpn. The -p flag shows the process name and PID.

Q2: How do you find which process is using port 443?

Answer: lsof -i :443 or ss -tulpn | grep :443

Q3: A server is slow. What command shows real-time CPU usage by process?

Answer: top or htop. Inside top, press 'P' to sort by CPU usage.

Q4: Where do you find failed SSH login attempts on a Debian/Ubuntu system?

Answer: /var/log/auth.log. Use grep "Failed password" /var/log/auth.log

Q5: What's the quickest way to block an IP that's brute-forcing SSH?

Answer: iptables -A INPUT -s [IP] -j DROP or install fail2ban for automatic blocking.

🧪 Lab Exercise: Investigate a Suspicious Server

Goal: Use Linux commands to investigate a potentially compromised server.

Scenario:

Your server is running slowly. Unknown connections spotted. Investigate!

Step 1: Check active connections

ss -tulpn
lsof -i -n

Step 2: Look for suspicious processes

ps aux --sort=-%cpu | head -10
top

Step 3: Check authentication logs

sudo cat /var/log/auth.log | grep "Failed password" | tail -20
lastb -10

Step 4: Review scheduled tasks (persistence)

crontab -l
ls -la /etc/cron*

🎓 Key Takeaways (Linux Security Commands)

  • ✅ Network: ss, lsof — find listening ports and active connections
  • ✅ Processes: ps, top — find suspicious processes, high CPU usage
  • ✅ Logs: journalctl, /var/log/auth.log — track failed logins, system events
  • ✅ File system: find with -perm — locate SUID files, world-writable files
  • ✅ Firewall: iptables (legacy) / ufw / firewalld — block attackers
  • ✅ Hardening: Disable root SSH, use fail2ban, keep packages updated
  • ✅ Always verify! Trust but verify — use hashes to check file integrity
