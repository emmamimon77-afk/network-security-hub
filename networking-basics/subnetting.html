✂️ Subnetting — Complete Guide

Subnetting = Dividing a large network into smaller, manageable subnetworks. Essential for IP address efficiency, security, and routing.

🎯 Why Subnetting Matters

  • ✅ Conserves IP addresses — Don't waste scarce IPv4 addresses
  • ✅ Improves security — Isolate departments (HR, Finance, Guest)
  • ✅ Reduces broadcast traffic — Less noise on each subnet
  • ✅ Simplifies troubleshooting — Smaller failure domains
💡 Real-world: A company with 200 devices doesn't need a /16 (65,534 addresses). Subnetting gives them a /24 (254 addresses) — efficient and secure.

📝 CIDR Notation (Classless Inter-Domain Routing)

CIDR = IP address / prefix-length (number of network bits)

CIDRSubnet MaskTotal IPsUsable IPsCommon Use
/8255.0.0.016,777,21616,777,214HUGE networks (rare)
/16255.255.0.065,53665,534Large organizations
/24255.255.255.0256254Standard LAN
/25255.255.255.128128126Small department
/26255.255.255.1926462Smaller group
/27255.255.255.2243230Very small team
/28255.255.255.2401614Point-to-point links
/30255.255.255.25242Router-to-router links
/32255.255.255.25511Single host (loopback)
📌 Formula: Total IPs = 2^(32 - prefix). Usable = Total - 2 (network + broadcast).

🎯 Finding Network Address, Broadcast, First & Last Host

Example: 192.168.1.45 /24

  • Network address: 192.168.1.0 (all host bits 0)
  • First usable host: 192.168.1.1
  • Last usable host: 192.168.1.254
  • Broadcast address: 192.168.1.255 (all host bits 1)

Example: 10.0.0.100 /26 (mask 255.255.255.192)

  • Blocks of 64 addresses (0-63, 64-127, 128-191, 192-255)
  • Network: 10.0.0.64
  • First host: 10.0.0.65
  • Last host: 10.0.0.126
  • Broadcast: 10.0.0.127
💡 Quick trick: Find the block size = 256 - octet value. For /26 (255.255.255.192), block = 256 - 192 = 64.

🏢 VLSM (Variable Length Subnet Mask) — Real Company Example

You have 192.168.1.0/24 (256 addresses). Assign subnets to:

  • 📍 HR: 50 devices → needs /26 (64 addresses)
  • 📍 IT: 30 devices → needs /27 (32 addresses)
  • 📍 Marketing: 20 devices → needs /27 (32 addresses)
  • 📍 Finance: 10 devices → needs /28 (16 addresses)
  • 📍 Guest Wi-Fi: 50 devices → needs /26 (64 addresses)

📝 Practice Questions (Test Yourself)

Q1: What is the subnet mask for /20?

Answer: 255.255.240.0 (16 + 4 = 20 bits, so 11111111.11111111.11110000.00000000)

Q2: 192.168.5.35/28 — What is the network address?

Answer: 192.168.5.32 (block size 16: 0,16,32,48 → 32-47 range)

Q3: How many usable IPs in a /29?

Answer: 2^(32-29)=8 total, minus 2 = 6 usable addresses.

Q4: What CIDR gives subnet mask 255.255.254.0?

Answer: /23 (255.255.254.0 = 23 network bits)

Q5: A /26 subnet — what are the valid host ranges in 10.0.0.0/26?

Answer: 10.0.0.1 - 10.0.0.62, 10.0.0.65 - 10.0.0.126, 10.0.0.129 - 10.0.0.190, 10.0.0.193 - 10.0.0.254

🏗️ Strategic Network Design: How to Plan Subnets for Real Companies

Stop guessing — learn the systematic method used by network architects.

📊 Step 1: Gather Requirements

Ask these questions before touching a calculator:

  • 📌 How many devices NOW? (workstations, servers, printers, IoT)
  • 📈 Growth rate? (20% per year? Double in 2 years?)
  • 🔒 Security zones? (HR separate from Guest? Finance isolated?)
  • 🌍 Physical locations? (One building? Multiple cities?)
  • 💰 Available IP space? (Do you have a /24, /20, or /16?)

🧮 Step 2: The "Powers of 2" Method (Find Smallest Usable Mask)

💡 Formula: 2^(host bits) - 2 ≥ Required hosts
Host bits = 32 - CIDR prefix

Example: Department needs 50 hosts

  • Try /26: 2^(6) - 2 = 64 - 2 = 62 hosts ✅ (fits, but 12 wasted)
  • Try /27: 2^(5) - 2 = 32 - 2 = 30 hosts ❌ (too small)
  • Winner: /26 (62 usable, room for 20% growth)

🗺️ Step 3: Hierarchical Addressing (The Professional Way)

Instead of random subnets, build a hierarchy:

Organization: 10.0.0.0/16 (65k addresses) │ ├── Site A (10.0.0.0/18) → 16k addresses │ ├── HR (10.0.0.0/22) │ ├── IT (10.0.4.0/22) │ └── Sales (10.0.8.0/22) │ ├── Site B (10.0.64.0/18) │ ├── HR (10.0.64.0/22) │ └── Manufacturing (10.0.68.0/22) │ └── Site C (10.0.128.0/18) └── DR Backup (10.0.128.0/21)

📐 Step 4: The "Biggest-First" Allocation Rule

Always assign subnets to the largest requirements first. This prevents fragmentation.

📝 Exercise: Design a network for:
• Finance: 100 hosts
• HR: 30 hosts
• IT: 15 hosts
• Guest Wi-Fi: 200 hosts
• Available space: 192.168.10.0/24

🎯 Show solution (largest first) 1. Guest (200 hosts) → /24 not possible (256 total). Need larger block or separate /24.
2. Finance (100 hosts) → /25 (126 usable): 192.168.10.0/25
3. HR (30 hosts) → /27 (30 usable): 192.168.10.128/27
4. IT (15 hosts) → /27 (30 usable): 192.168.10.160/27
Note: Guest Wi-Fi would need its own separate /24 or use a different private range (172.16.0.0/12).

⚖️ Step 5: Right-Sizing vs. Future Growth

  • ✅ Conservative: Exact fit + 10% — minimal waste
  • ✅ Aggressive: Exact fit — risky if growth happens
  • ✅ Liberal: Double the size — 50% waste but safe
⚠️ Real-world advice: Always leave 20-30% free in each subnet. Running out of IPs is a painful redesign.

💼 Real-World Case Study: Startup Growth

Scenario: Startup has 192.168.1.0/24. Currently 20 employees. Projecting 100 in 12 months.

  • ❌ Bad design: /27 per department (wasteful, complex)
  • ✅ Good design: Single /24 for now, but plan to migrate to 10.0.0.0/16
  • ✅ Best design: Start with 10.0.0.0/22 for HQ, use /24 per floor, /27 per team

🎓 Key Takeaways (Subnetting Essentials)

  • ✅ Subnetting = Borrow bits from host portion to create smaller networks
  • ✅ Formula: Total IPs = 2^(32 - prefix), Usable = Total - 2
  • ✅ Block size = 256 - last non-zero octet in mask (for /24 to /32)
  • ✅ VLSM = Different subnets per department (start with largest)
  • ✅ Practice order: Binary → CIDR → Block size → Network/Broadcast → VLSM
