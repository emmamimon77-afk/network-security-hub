⚙️ Cisco Commands — Complete CLI Guide

Cisco IOS (Internetwork Operating System) — The industry-standard CLI for routers and switches. Master these commands for CCNA, network admin, and security roles.

🎯 Why Cisco Commands Matter

  • ✅ Industry standard — 80%+ of enterprise networks run Cisco
  • ✅ CCNA/CCNP certification — CLI mastery is required
  • ✅ Real troubleshooting — Most network issues are fixed in CLI
  • ✅ Transferable skills — Other vendors (Juniper, Arista) use similar concepts
💡 Real-world: When a router crashes or a switch floods, GUI is useless. CLI saves the day.

🔐 Cisco CLI Modes (The Hierarchy)

Every Cisco device has multiple security levels. You must climb the ladder:

┌─────────────────────────────────────────────────────────┐ │ User EXEC Mode (>) │ │ └── Limited viewing, no changes │ │ │ │ │ │ enable │ │ ▼ │ │ Privileged EXEC Mode (#) │ │ └── Full viewing, some testing (ping, traceroute) │ │ │ │ │ │ configure terminal │ │ ▼ │ │ Global Configuration Mode (config)# │ │ └── Changes affect entire device │ │ │ │ │ │ interface gigabit0/1 │ │ ▼ │ │ Interface Configuration Mode (config-if)# │ │ └── Changes affect only that port │ └─────────────────────────────────────────────────────────┘

👁️ Essential Show Commands (View Configuration)

Show commands never change anything — they just display information. Always start here when troubleshooting.

CommandWhat It ShowsExample Output
show running-config Current active configuration (RAM) All settings since last boot
show startup-config Saved configuration (NVRAM) What loads on next reboot
show ip interface brief All interfaces with IPs and status Gig0/0: 192.168.1.1, up/up
show interfaces Detailed stats for all interfaces Packets, errors, speed, duplex
show vlan brief VLANs and assigned ports VLAN 10: Gig0/1-24
show mac address-table MAC addresses learned by switch MAC → Port mapping
show ip route Routing table Networks and next hops
show version Device info, uptime, IOS version Model, RAM, flash, uptime
show cdp neighbors Directly connected Cisco devices Neighbor device names and ports

⚙️ Configuration Commands (Making Changes)

Changes are active immediately but not saved unless you write them.

Basic Device Configuration

Router> enable
Router# configure terminal
Router(config)# hostname BRANCH-ROUTER
BRANCH-ROUTER(config)# banner motd ^C Unauthorized access prohibited ^C
BRANCH-ROUTER(config)# enable secret Cisco123
BRANCH-ROUTER(config)# line console 0
BRANCH-ROUTER(config-line)# password admin123
BRANCH-ROUTER(config-line)# login

Interface Configuration (IP Address)

Router(config)# interface gigabitEthernet 0/0
Router(config-if)# ip address 192.168.1.1 255.255.255.0
Router(config-if)# no shutdown
Router(config-if)# description Link to Main Office

VLAN Configuration (Switch)

Switch(config)# vlan 10
Switch(config-vlan)# name Engineering
Switch(config-vlan)# exit
Switch(config)# interface range gigabitEthernet 0/1-24
Switch(config-if-range)# switchport mode access
Switch(config-if-range)# switchport access vlan 10

Static Route Configuration

Router(config)# ip route 192.168.2.0 255.255.255.0 10.0.0.2
Router(config)# ip route 0.0.0.0 0.0.0.0 10.0.0.1 # Default route

Saving Configuration

Router# copy running-config startup-config
Or:
Router# write memory

🌐 Real-World Scenarios (What Would You Do?)

Scenario 1: New switch, need to configure VLANs for HR and IT.
Solution:
enable → conf t → vlan 20 → name HR → exit → vlan 30 → name IT → exit → interface range gig0/1-12 → switchport mode access → switchport access vlan 20 → exit → interface range gig0/13-24 → switchport mode access → switchport access vlan 30
Scenario 2: Router interface down — how to check and fix?
Troubleshooting steps:
1. show ip interface brief (find down interface)
2. show interface gig0/0 (check for errors)
3. interface gig0/0 → no shutdown (turn on)
4. end → copy run start
Scenario 3: Can't remember if you saved config before reboot.
Solution:
show running-config vs show startup-config — compare timestamps or use show version (last reload reason).

📝 Practice Questions (Cisco-Style)

Q1: What command moves from User EXEC to Privileged EXEC?

Answer: enable

Q2: You're in interface config mode. How do you return to Privileged EXEC immediately?

Answer: end or Ctrl+Z

Q3: Which command shows all interfaces with IP addresses and status?

Answer: show ip interface brief

Q4: You configured a new VLAN but it's not working after reboot. Why?

Answer: You need to save with copy running-config startup-config or write memory.

Q5: What's the difference between show running-config and show startup-config?

Answer: Running = current active config (RAM). Startup = saved config (NVRAM) loaded on reboot.

🧪 Lab Exercise: Build a Small Office Network

Goal: Configure a router and switch using Cisco CLI commands.

📡 Step 1 – Access CLI (Packet Tracer or Real Device)

If you don't have equipment, download Cisco Packet Tracer (free for Cisco NetAcad members).

🔢 Step 2 – Basic Router Setup

Router> enable
Router# configure terminal
Router(config)# hostname OFFICE-ROUTER
OFFICE-ROUTER(config)# enable secret class123
OFFICE-ROUTER(config)# line console 0
OFFICE-ROUTER(config-line)# password cisco123
OFFICE-ROUTER(config-line)# login
OFFICE-ROUTER(config-line)# exit

🖥️ Step 3 – Configure Interface

OFFICE-ROUTER(config)# interface gig0/0
OFFICE-ROUTER(config-if)# ip address 192.168.1.1 255.255.255.0
OFFICE-ROUTER(config-if)# no shutdown
OFFICE-ROUTER(config-if)# description LAN Interface
OFFICE-ROUTER(config-if)# exit

🌍 Step 4 – Add Default Route

OFFICE-ROUTER(config)# ip route 0.0.0.0 0.0.0.0 10.0.0.1

📝 Step 5 – Save Configuration

OFFICE-ROUTER# copy running-config startup-config
💡 Challenge: After saving, reboot the router (reload) and verify your config is still there with show running-config.

🎓 Key Takeaways (Cisco CLI Mastery)

  • ✅ CLI modes ladder: User EXEC → Privileged EXEC → Global Config → Interface Config
  • ✅ Enable vs Enable Secret: enable secret is encrypted, always use it
  • ✅ Show commands never change config — safe for troubleshooting
  • ✅ Always save: copy running-config startup-config or config lost on reboot
  • ✅ Basic pattern: conf t → changes → exit → write memory
  • ✅ Context-sensitive help: ? shows available commands, show ip ? narrows down
