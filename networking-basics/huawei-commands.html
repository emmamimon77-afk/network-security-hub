⚙️ Huawei Commands — Complete Guide

For Cisco professionals transitioning to Huawei. This lesson provides side-by-side comparisons of Cisco and Huawei CLI commands. Concepts are identical — only the syntax changes.

💡 Prerequisite: Complete the Cisco Commands lesson first. This lesson assumes you understand networking concepts and focuses on the differences.

🔐 CLI Modes Comparison

ModeCiscoHuaweiPrompt Example
User View No command needed (default) No command needed (default) <Huawei>
Privileged View enable system-view [~Huawei] (if unsaved) or [Huawei] (saved)
Interface View interface gig0/0 interface gigabitEthernet 0/0/0 [Huawei-GigabitEthernet0/0/0]

📊 Cisco vs Huawei Command Comparison

PurposeCisco CommandHuawei Command
View current configuration show running-config display current-configuration
View saved configuration show startup-config display saved-configuration
View interface status show ip interface brief display ip interface brief
View routing table show ip route display ip routing-table
View VLANs show vlan brief display vlan

Exit current mode exit quit

Negate/remove command no [command] undo [command]

⚙️ Basic Device Configuration — Side by Side

Cisco

Router> enable
Router# configure terminal
Router(config)# hostname BRANCH-ROUTER
BRANCH-ROUTER(config)# banner motd ^C Authorized Access Only ^C
BRANCH-ROUTER(config)# enable secret Cisco123
BRANCH-ROUTER(config)# line console 0
BRANCH-ROUTER(config-line)# password admin123
BRANCH-ROUTER(config-line)# login
BRANCH-ROUTER(config-line)# exit
BRANCH-ROUTER(config)# exit
BRANCH-ROUTER# write memory

Huawei

<Huawei> system-view
[~Huawei] sysname BRANCH-ROUTER
[~BRANCH-ROUTER] header login information "Authorized Access Only"
[~BRANCH-ROUTER] aaa
[~BRANCH-ROUTER-aaa] local-user admin password cipher admin123
[~BRANCH-ROUTER-aaa] local-user admin privilege level 15
[~BRANCH-ROUTER-aaa] local-user admin service-type terminal
[~BRANCH-ROUTER-aaa] quit
[~BRANCH-ROUTER] user-interface console 0
[~BRANCH-ROUTER-ui-console0] authentication-mode aaa
[~BRANCH-ROUTER-ui-console0] quit
[~BRANCH-ROUTER] return
<BRANCH-ROUTER> save
💡 Key Differences:
  • Huawei uses system-view to enter privileged mode (not enable)
  • Huawei uses sysname (not hostname)
  • Huawei uses undo (not no)
  • Huawei uses quit (not exit)
  • Huawei uses return (not end)
  • Huawei uses save (not write memory)

🔌 Interface Configuration — Side by Side

Cisco

Router(config)# interface gigabitEthernet 0/0
Router(config-if)# ip address 192.168.1.1 255.255.255.0
Router(config-if)# no shutdown
Router(config-if)# description Link to Main Office
Router(config-if)# exit

Huawei

[~BRANCH-ROUTER] interface gigabitEthernet 0/0/0
[~BRANCH-ROUTER-GigabitEthernet0/0/0] ip address 192.168.1.1 24
[~BRANCH-ROUTER-GigabitEthernet0/0/0] undo shutdown
[~BRANCH-ROUTER-GigabitEthernet0/0/0] description Link to Main Office
[~BRANCH-ROUTER-GigabitEthernet0/0/0] quit
💡 Key Differences:
  • Huawei interface naming: gigabitEthernet 0/0/0 (slot/subslot/port) vs Cisco gig0/0
  • Huawei subnet mask can be written as 24 instead of 255.255.255.0
  • Huawei uses undo shutdown (not no shutdown)

🏷️ VLAN Configuration — Side by Side

Cisco

Switch(config)# vlan 10
Switch(config-vlan)# name Engineering
Switch(config-vlan)# exit
Switch(config)# interface range gigabitEthernet 0/1-12
Switch(config-if-range)# switchport mode access
Switch(config-if-range)# switchport access vlan 10
Switch(config-if-range)# no shutdown

Huawei

[~Switch] vlan batch 10
[~Switch] vlan 10
[~Switch-vlan10] description Engineering
[~Switch-vlan10] quit
[~Switch] interface gigabitEthernet 0/0/1 to 0/0/12
[~Switch-GigabitEthernet0/0/1] port link-type access
[~Switch-GigabitEthernet0/0/1] port default vlan 10
[~Switch-GigabitEthernet0/0/1] undo shutdown
💡 Key Differences:
  • Huawei uses vlan batch 10 to create multiple VLANs at once
  • Huawei uses port link-type access (not switchport mode access)
  • Huawei uses port default vlan 10 (not switchport access vlan 10)

🗺️ Static Route Configuration — Side by Side

Cisco

Router(config)# ip route 192.168.2.0 255.255.255.0 10.0.0.2
Router(config)# ip route 0.0.0.0 0.0.0.0 10.0.0.1
Router(config)# ip route 172.16.0.0 255.255.0.0 serial0/0/0

Huawei

[~Router] ip route-static 192.168.2.0 24 10.0.0.2
[~Router] ip route-static 0.0.0.0 0.0.0.0 10.0.0.1
[~Router] ip route-static 172.16.0.0 16 serial 0/0/0
💡 Key Differences:
  • Huawei uses ip route-static (not ip route)
  • Huawei can use CIDR notation (24 instead of 255.255.255.0)
  • Huawei interface syntax: serial 0/0/0 (not serial0/0/0)

🔄 OSPF Configuration — Side by Side

Cisco

Router(config)# router ospf 1
Router(config-router)# router-id 1.1.1.1
Router(config-router)# network 192.168.1.0 0.0.0.255 area 0
Router(config-router)# network 10.0.0.0 0.0.0.3 area 0
Router(config-router)# passive-interface gig0/0

Huawei

[~Router] ospf 1 router-id 1.1.1.1
[~Router-ospf-1] area 0
[~Router-ospf-1-area-0.0.0.0] network 192.168.1.0 0.0.0.255
[~Router-ospf-1-area-0.0.0.0] network 10.0.0.0 0.0.0.3
[~Router-ospf-1-area-0.0.0.0] quit
[~Router-ospf-1] silent-interface gigabitEthernet 0/0/0
💡 Key Differences:
  • Huawei configures OSPF in a single command: ospf 1 router-id 1.1.1.1
  • Huawei uses area 0 sub-mode (Cisco includes area in network statement)
  • Huawei uses silent-interface (not passive-interface)

🧪 Lab Exercise: Getting Started with eNSP (Huawei Simulator)

eNSP (Enterprise Network Simulation Platform) is Huawei's free network simulator — equivalent to Cisco Packet Tracer.

Step 1: Download and Install eNSP

Download from Huawei's official site (requires free registration):
https://support.huawei.com/enterprise/en/ensp

Installation includes:
- eNSP itself
- WinPcap (network driver)
- Wireshark (packet capture)
- VirtualBox (for router/switch emulation)

Step 2: Create a Simple Topology

1. Open eNSP → New Topology
2. Add two AR2220 routers
3. Connect with Ethernet cable
4. Right-click routers → Start

Step 3: Configure Basic Huawei CLI

<Huawei> system-view
[~Huawei] sysname R1
[~R1] interface gigabitEthernet 0/0/0
[~R1-GigabitEthernet0/0/0] ip address 10.0.0.1 24
[~R1-GigabitEthernet0/0/0] undo shutdown
[~R1-GigabitEthernet0/0/0] quit
[~R1] return
<R1> save
💡 Challenge: Add a second router, configure IPs in the same subnet, and ping between them. Then add static routes to reach non-directly connected networks.

📝 Practice Questions

Q1: What is the Huawei equivalent of Cisco's show running-config?
Answer: display current-configuration
Q2: How do you enter privileged view on a Huawei device?
Answer: system-view (Cisco uses enable)
Q3: What command saves the configuration on a Huawei router?
Answer: save (Cisco uses write memory or copy running-config startup-config)
Q4: How do you negate a command on Huawei?
Answer: undo [command] (Cisco uses no [command])
Q5: What is the Huawei equivalent of Cisco's no shutdown?
Answer: undo shutdown

🎓 Key Takeaways (Cisco → Huawei Migration)

  • ✅ Concepts are identical — only syntax changes
  • ✅ show → display (e.g., show ip route → display ip routing-table)
  • ✅ no → undo (e.g., no shutdown → undo shutdown)
  • ✅ enable → system-view (privileged mode entry)
  • ✅ exit → quit (exit current mode)
  • ✅ end → return (return to user view)
  • ✅ write memory → save (save configuration)
  • ✅ ip route → ip route-static (static routes)
  • ✅ interface naming: Cisco gig0/0 → Huawei gigabitEthernet 0/0/0
