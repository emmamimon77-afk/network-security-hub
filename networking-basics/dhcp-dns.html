🌐 DHCP & DNS — Complete Guide

Dynamic Host Configuration Protocol (DHCP) automatically assigns IP addresses to devices. Domain Name System (DNS) translates domain names (google.com) into IP addresses (142.250.185.46). Together, they make networks "just work."

💡 Prerequisite: Basic IP addressing. DHCP and DNS are everywhere — from home routers to enterprise servers.

📡 Part 1: DHCP — Dynamic Host Configuration Protocol

Why DHCP Matters

  • ✅ No manual IP configuration — Plug and play
  • ✅ Prevents IP conflicts — Server manages unique addresses
  • ✅ Centralized management — Change DNS servers, gateways from one place
  • ✅ Efficient IP usage — Leases return to pool when device leaves

DHCP DORA Process (4 Steps)

Client DHCP Server │ │ │ 1. DISCOVER (broadcast) │ │ "Is there a DHCP server?" │ │─────────────────────────────▶│ │ │ │ 2. OFFER (unicast/broadcast) │ │ "I can offer 192.168.1.100" │ │◀─────────────────────────────│ │ │ │ 3. REQUEST (broadcast) │ │ "I'll take 192.168.1.100" │ │─────────────────────────────▶│ │ │ │ 4. ACK (unicast) │ │ "Approved. Lease duration 24h"│ │◀─────────────────────────────│ │ │

DHCP Lease Timers

  • T1 (50% of lease): Client tries to renew with original server
  • T2 (87.5% of lease): Client broadcasts to any DHCP server if T1 failed
  • Lease end: IP returns to pool, client must request new

DHCP Configuration (Cisco Router)

! Exclude static IPs (servers, printers, routers)
Router(config)# ip dhcp excluded-address 192.168.1.1 192.168.1.50

! Create DHCP pool
Router(config)# ip dhcp pool LAN_POOL
Router(dhcp-config)# network 192.168.1.0 255.255.255.0
Router(dhcp-config)# default-router 192.168.1.1
Router(dhcp-config)# dns-server 8.8.8.8 8.8.4.4
Router(dhcp-config)# lease 7 0 0 (7 days)
Router(dhcp-config)# domain-name mycompany.local
Router(dhcp-config)# exit

! Verify DHCP
Router# show ip dhcp binding
Router# show ip dhcp pool
Router# debug ip dhcp server events

DHCP Relay (IP Helper)

When DHCP server is in a different subnet, you need a relay agent.

Router(config)# interface vlan 10
Router(config-if)# ip helper-address 10.0.0.5 (DHCP server IP)

DHCP Security Risks

  • Rogue DHCP servers — Malicious devices handing out bad IPs
  • DHCP starvation — Attacker exhausts all IP addresses
  • Mitigation: DHCP Snooping (Cisco feature)

🔍 Part 2: DNS — Domain Name System

Why DNS Matters

  • ✅ Human-friendly names — Remember google.com, not 142.250.185.46
  • ✅ Load balancing — One name can map to many IPs
  • ✅ Location independence — Change server IPs without changing names

DNS Hierarchy

[Root] (.) │ ┌───────┴───────┐ [.com] [.org] (Top-Level Domains) │ ┌────┴────┐ [google] [cisco] (Second-Level Domains) │ ┌────┴────┐ [www] [mail] (Subdomains)

DNS Record Types

RecordPurposeExample
A IPv4 address google.com → 142.250.185.46
AAAA IPv6 address google.com → 2001:4860:4860::8888
CNAME Alias (name → another name) www.google.com → google.com
MX Mail exchanger (email routing) mail.google.com (priority 10)
NS Name server (authoritative DNS servers) ns1.google.com, ns2.google.com
PTR Reverse lookup (IP → name) 142.250.185.46 → google.com

DNS Resolution Process

Browser → Local Cache → Hosts File → DNS Resolver (ISP/8.8.8.8) → Root Server → TLD Server → Authoritative Server → Response

DNS Configuration (Cisco Router as DNS Server)

! Enable DNS server on router
Router(config)# ip dns server

! Define hostname-to-IP mappings
Router(config)# ip host web-server 192.168.1.10
Router(config)# ip host mail-server 192.168.1.20

! Configure DNS forwarders (for unknown domains)
Router(config)# ip name-server 8.8.8.8
Router(config)# ip name-server 8.8.4.4

! Verify DNS
Router# show hosts
Router# ping web-server

DNS Security

  • DNS spoofing/cache poisoning — Attacker injects fake DNS entries
  • DNSSEC — Digitally signs DNS responses (prevents spoofing)
  • DNS over TLS (DoT) / DNS over HTTPS (DoH) — Encrypts DNS queries
  • Best practice: Use trusted DNS servers (Cloudflare 1.1.1.1, Google 8.8.8.8)

📝 Practice Questions

Q1: What are the four steps of the DHCP DORA process?
Answer: Discover → Offer → Request → Acknowledgment (ACK)
Q2: What DNS record type maps a domain name to an IPv4 address?
Answer: A record (AAAA is for IPv6)
Q3: A DHCP server is in a different subnet than the clients. What feature is needed?
Answer: DHCP relay (ip helper-address on Cisco routers)
Q4: What is the purpose of DNSSEC?
Answer: DNSSEC digitally signs DNS responses to prevent cache poisoning and spoofing attacks.
Q5: What command shows active DHCP leases on a Cisco router?
Answer: show ip dhcp binding

🎓 Key Takeaways (DHCP & DNS Mastery)

  • ✅ DHCP DORA — Discover, Offer, Request, ACK
  • ✅ DHCP Relay — ip helper-address for cross-subnet DHCP
  • ✅ DNS Hierarchy — Root → TLD → Authoritative
  • ✅ Common DNS records — A (IPv4), AAAA (IPv6), CNAME (alias), MX (mail)
  • ✅ DHCP Snooping — Prevents rogue DHCP servers
  • ✅ DNSSEC / DoH / DoT — DNS security mechanisms
