📚 The OSI Model — Complete Guide

Open Systems Interconnection Model | A 7-layer framework that standardizes network communication.

🎯 Why the OSI Model Still Matters

  • 🔍 Troubleshoot systematically — "Which layer is failing?"
  • 🛡️ Understand security attacks — Each layer has unique threats
  • 💬 Communicate with other engineers — Common reference frame
💡 Interview Tip: "Explain what happens when you type google.com" — walk down OSI layers!

📊 The 7 Layers — Detailed

Layer 7ApplicationHTTP, HTTPS, FTP, SSH, DNS, SMTP
Layer 6PresentationTLS/SSL encryption, JPEG, compression
Layer 5SessionNetBIOS, RPC, session management
Layer 4TransportTCP (reliable), UDP (fast), ports 0-65535
Layer 3NetworkIPv4, IPv6, ICMP (ping), routing (OSPF, BGP)
Layer 2Data LinkMAC addresses, Ethernet, switches, VLANs
Layer 1PhysicalCat5e/6, fiber, wireless, voltage, bits

📦 Encapsulation (How Data Travels)

Layer 7: [Data "Hello"] ⬇️ Layer 4: [TCP Header] + [Data] = Segment ⬇️ Layer 3: [IP Header] + [Segment] = Packet ⬇️ Layer 2: [MAC Header] + [Packet] + [Trailer] = Frame ⬇️ Layer 1: Bits on wire: 101100111000...

🛠️ Troubleshooting Scenarios

❌ "Cannot ping 8.8.8.8 but can ping 192.168.1.1"

Layer 3 issue — gateway works but internet routing is broken. Check default route: ip route show

❌ "Wi-Fi connected but no internet"

Layer 2 works, Layer 3 fails. Renew DHCP: sudo dhclient (Linux)

📝 Practice Questions

Q1: Which layer handles IP addresses?

Layer 3 — Network Layer

Q2: What's the difference between a packet and a frame?

Packet = Layer 3 (IP header + data). Frame = Layer 2 (MAC header + packet + trailer).
