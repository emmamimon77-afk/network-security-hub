📚 The OSI Model

Open Systems Interconnection Model — A conceptual framework used to understand network communication in 7 layers.

🧠 Why Learn the OSI Model?

The OSI model helps you:

  • ✅ Troubleshoot networks layer by layer
  • ✅ Understand how data travels from an app to another device
  • ✅ Speak the language of network engineers and security analysts

📊 The 7 Layers of OSI (Layer 7 to Layer 1)

Layer 7
Application
HTTP, FTP, SMTP, DNS — where users interact
Layer 6
Presentation
Encryption, compression, translation (TLS, JPEG)
Layer 5
Session
Establishes, manages, ends sessions (NetBIOS, RPC)
Layer 4
Transport
TCP / UDP — segmentation, reliability, flow control
Layer 3
Network
IP, routing, logical addressing (Routers)
Layer 2
Data Link
MAC addresses, switches, framing (Ethernet)
Layer 1
Physical
Cables, fiber, radio signals, voltage

📝 Data Flow: How a Message Travels

+-------------+ | Layer 7 | "Hello" (Application data) +-------------+ ⬇️ (encapsulation) +-------------+ | Layer 4 | TCP segment (adds port numbers) +-------------+ ⬇️ +-------------+ | Layer 3 | IP packet (adds source/dest IP) +-------------+ ⬇️ +-------------+ | Layer 2 | Ethernet frame (adds MAC addresses) +-------------+ ⬇️ +-------------+ | Layer 1 | Bits on the wire (1s and 0s) +-------------+

At the receiving device: The process reverses (de-encapsulation) — each layer strips off its header and passes the data up.
