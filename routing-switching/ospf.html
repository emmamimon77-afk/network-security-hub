🔄 OSPF — Complete Guide

Open Shortest Path First (OSPF) = A dynamic link-state routing protocol used by routers to automatically find the best path to destinations. Industry standard for enterprise networks.

🎯 Why OSPF Matters

  • ✅ Industry standard — Works across Cisco, Juniper, Huawei, Arista
  • ✅ Fast convergence — Reacts to failures in seconds (not minutes)
  • ✅ Scalable — Supports thousands of routers via areas
  • ✅ No loop issues — Link-state protocols inherently loop-free
  • ✅ Vendor neutral — Open standard (unlike EIGRP, which is Cisco proprietary)
💡 Real-world: Your ISP, cloud providers (AWS/Azure), and most large enterprises run OSPF internally.

📊 OSPF vs Static Routing (When to Use Each)

FeatureStatic RoutingOSPF (Dynamic)

📌 Rule of thumb: Static for stub networks (branch offices). OSPF for core/backbone networks.

📚 Core OSPF Concepts (Must Know)

1. Router ID (RID)

Unique 32-bit identifier for each OSPF router. Used to track neighbors.

Selection order:
1. Manually configured: router-id 1.1.1.1
2. Highest IP on loopback interface
3. Highest IP on physical interface

2. Areas

Logical grouping of routers to reduce routing table size and LSA flooding.

  • Area 0 (Backbone): Connects all other areas — must exist
  • Standard areas: Any number (1-4 billion)
  • Stub areas: Block external routes (reduces table size)

3. Hello Protocol

Routers send Hello packets to discover and maintain neighbors.

  • Default Hello interval: 10 seconds (broadcast networks)
  • Dead interval: 40 seconds (4× Hello)

4. Cost (Metric)

OSPF uses cost = reference bandwidth / interface bandwidth.
Lower cost = better path.

Default reference bandwidth: 100 Mbps
• FastEthernet (100 Mbps): cost = 100/100 = 1
• GigabitEthernet (1000 Mbps): cost = 100/1000 = 1 (problem!)
• Fix: auto-cost reference-bandwidth 10000 (10 Gbps reference)

5. Link State Database (LSDB)

Every router builds the exact same map of the network using LSAs.

⚙️ OSPF Configuration (Cisco CLI)

Basic OSPF on Single Router

Router(config)# router ospf 1
Router(config-router)# router-id 1.1.1.1
Router(config-router)# network 192.168.1.0 0.0.0.255 area 0
Router(config-router)# network 10.0.0.0 0.0.0.3 area 0
Router(config-router)# passive-interface gigabitEthernet 0/0
Router(config-router)# end

Understanding Wildcard Masks (Inverse Mask)

OSPF uses wildcard masks = 255.255.255.255 - subnet mask.

• Subnet 192.168.1.0/24 → Wildcard: 0.0.0.255
• Subnet 10.0.0.0/30 → Wildcard: 0.0.0.3
• Single IP (host) → Wildcard: 0.0.0.0

Verification Commands

Router# show ip ospf neighbor
Router# show ip ospf interface
Router# show ip route ospf
Router# show ip ospf database

🔄 OSPF Operation (How It Works)

Step-by-Step Process:

  1. Discover neighbors → Send Hello packets (multicast 224.0.0.5)
  2. Establish adjacency → Exchange database descriptions
  3. Synchronize LSDB → Request/update missing LSAs
  4. Run SPF algorithm → Calculate shortest paths (Dijkstra)
  5. Populate routing table → Install best routes
  6. Maintain neighbors → Periodic Hellos (10 sec)
  7. React to changes → Triggered updates on failure

LSA Types (For CCNA/Exams)

TypeNameWhat It Contains
1Router LSARouter's links within its area
2Network LSAList of routers on a broadcast segment (DR sends)
3Summary LSARoutes from one area to another (ABR sends)
4ASBR SummaryRoute to ASBR
5External LSARoutes redistributed from other protocols (ASBR sends)

🛡️ OSPF Security

Authentication Types

  • Null (Type 0): No authentication — default, insecure
  • Plain text (Type 1): Password in clear text — avoid
  • MD5 (Type 2): Hashed password — recommended
  • SHA (Type 3): Stronger hashing — modern networks
Enable MD5 Authentication:
Router(config)# interface gigabitEthernet 0/0
Router(config-if)# ip ospf authentication message-digest
Router(config-if)# ip ospf message-digest-key 1 md5 SecurePass123
⚠️ Always authenticate OSPF! Without authentication, attackers can inject fake routes and redirect traffic.

⚖️ OSPF Load Balancing Equal-Cost Multi-Path (ECMP)

When OSPF finds multiple paths with identical cost, it load balances across all of them.

                    ┌──────────┐
                    │ Router1  │
                    └────┬──┬──┘
                         │  │
            Cost 10 ─────┘  └───── Cost 10
                         │  │
                    ┌────┴──┴──┐
                    │ Router2  │
                    └──────────┘
            
            Router1 can reach Router2 via two equal-cost links.
            OSPF installs BOTH routes!
Router# show ip route
O 192.168.2.0/24 [110/2] via 10.0.0.2, 00:00:01, Gig0/0
[110/2] via 10.0.0.6, 00:00:01, Gig0/1
💡 Default ECMP maximum: 4 paths (can increase to 16 with maximum-paths 16)

🔧 Troubleshooting OSPF

❌ Neighbors not forming?
Check:
1. Same area ID on both routers?
2. Same Hello/Dead timers?
3. Same authentication key if configured?
4. Interfaces in same subnet?
5. No ACL blocking 224.0.0.5?
❌ Routes missing from routing table?
Check:
1. Are routes in OSPF database? show ip ospf database
2. Stub area blocking external routes?
3. Passive-interface preventing Hellos?
4. Inbound filtering on neighbor?
5. Auto-cost reference bandwidth mismatch?
❌ Wrong path selected?
Check:
1. Interface costs (show ip ospf interface)
2. Reference bandwidth (auto-cost reference-bandwidth)
3. Manually set cost with ip ospf cost [value]

💼 Real-World Case Study: University Campus OSPF Design

Scenario: A university with 4 campuses (Main, North, South, East) wants OSPF for internal routing.

Requirements:

  • 7,500 students + 500 staff → 8,000 devices
  • Main campus = data center (servers, admin)
  • Remote campuses need internet via Main
  • Guest Wi-Fi should be isolated

Solution:

  • Area 0 (Backbone): Main campus core routers
  • Area 1,2,3: Remote campuses (stub areas)
  • Guest VLAN: Separate OSPF process or static route to firewall
  • Redundancy: Dual links from each campus to Main

                          ┌──────────────────┐
                          │   Main Campus    │
                          │   Core Routers   │
                          │  (Data Center)   │
                          └────┬──────┬──────┘
                               │      │
                ┌──────────────┘      └──────────────┐
                │ (Area 1 Stub)       (Area 2 Stub)  │
                ▼                                     ▼
          ┌──────────┐                         ┌──────────┐
          │  North   │                         │  South   │
          │ Campus   │                         │ Campus   │
          │ Branch1  │                         │ Branch2  │
          └──────────┘                         └──────────┘
💡 Result: Automatic failover, optimal routing, and reduced routing tables at branch campuses (stub areas block external routes).

📝 Practice Questions (CCNA-Style)

Q1: What command enables OSPF process 1 on a Cisco router?

Answer: router ospf 1 in global configuration mode.

Q2: What is the default Hello interval on a broadcast network?

Answer: 10 seconds.

Q3: A router has a GigabitEthernet interface and FastEthernet. Which has lower cost?

Answer: GigabitEthernet (1000 Mbps) has lower cost (1) than FastEthernet (100 Mbps, cost 1) — but only if reference bandwidth is set correctly (10000 for Gigabit to show difference).

Q4: What is the purpose of Area 0?

Answer: Backbone area — all other areas must connect through Area 0.

Q5: Which LSA type is flooded within an area and contains router links?

Answer: Type 1 (Router LSA).

Q6: A customer has 100+ routers and slow CPU. Which OSPF feature reduces routing table size?

Answer: Stub areas or totally stubby areas (block external or summary routes).

StandardOpen (RFC 2328)Cisco proprietary

Metric Cost (bandwidth) Composite (bandwidth + delay)

🎓 Key Takeaways (OSPF Mastery)

  • ✅ OSPF = Link-state routing protocol — maintains complete network map
  • ✅ Area 0 required — all areas connect through backbone
  • ✅ Cost = reference / bandwidth — lower cost = better path
  • ✅ Hello = 10 sec, Dead = 40 sec on broadcast networks
  • ✅ Wildcard masks = inverse of subnet mask
  • ✅ Stub areas reduce routing tables — block external routes
  • ✅ Always authenticate OSPF — prevent route injection
  • ✅ ECMP = Load balancing across equal-cost paths
