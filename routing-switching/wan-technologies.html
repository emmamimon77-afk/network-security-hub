🌍 WAN Technologies — Complete Guide

Wide Area Network (WAN) technologies connect networks across cities, countries, and continents. This lesson covers HDLC, PPP, GRE tunnels, Frame Relay, and MPLS — essential for enterprise networking and certification exams.

💡 Prerequisite: Basic routing concepts. WAN technologies are how branch offices connect to headquarters.

🌐 WAN Overview — Connecting Far Away Networks

┌─────────────┐ ┌─────────────┐ │ HQ Office │ │ Branch Office│ │ Router A │════════════════════│ Router B │ └─────────────┘ └─────────────┘ │ │ │ WAN Link (Serial) │ │ (Leased line, MPLS, etc.) │ └──────────────────────────────────┘
  • ✅ Connects geographically separated networks
  • ✅ Uses different technologies than LAN (serial links, not Ethernet)
  • ✅ Often provided by ISPs (Internet Service Providers)

🔗 HDLC — High-Level Data Link Control

HDLC is the default encapsulation on Cisco serial interfaces. It is a bit-oriented, synchronous protocol that uses frame flags and CRC error checking.

HDLC Characteristics:

  • ✅ Cisco proprietary extension — Standard HDLC ≠ Cisco HDLC
  • ✅ Default on Cisco serial interfaces — No configuration needed for basic operation
  • ✅ Point-to-point only — Does not support multiple connections
  • ✅ No authentication — Cannot verify peer identity

HDLC Configuration (Cisco):

Router(config)# interface serial 0/0/0
Router(config-if)# encapsulation hdlc (default, may not show)
Router(config-if)# ip address 10.0.0.1 255.255.255.252
Router(config-if)# clock rate 64000 (DCE side only)
Router(config-if)# no shutdown
📋 Verify HDLC:
Router# show interface serial 0/0/0 | include Encapsulation
💡 Note: Cisco HDLC is NOT compatible with other vendors. Use PPP for multi-vendor environments.

🔐 PPP — Point-to-Point Protocol

PPP is the industry standard for point-to-point links. It supports authentication (PAP/CHAP), multilink, and works with any vendor.

PPP Components:

  • LCP (Link Control Protocol): Establishes, configures, and tests the link
  • NCP (Network Control Protocol): Configures network layer protocols (IP, IPX, etc.)
  • Authentication: PAP (password) or CHAP (challenge-response)

PPP Configuration (Cisco):

Router(config)# interface serial 0/0/0
Router(config-if)# encapsulation ppp
Router(config-if)# ppp authentication chap
Router(config-if)# username BRANCH password Cisco123
Router(config-if)# ip address 10.0.0.1 255.255.255.252
Router(config-if)# no shutdown

PPP with PAP Authentication (Less Secure):

Router(config)# interface serial 0/0/0
Router(config-if)# encapsulation ppp
Router(config-if)# ppp authentication pap
Router(config-if)# ppp pap sent-username HQ password SecurePass

PPP Multilink (Combine Multiple Links):

Router(config)# interface multilink 1
Router(config-if)# ip address 10.0.0.1 255.255.255.252
Router(config-if)# ppp multilink
Router(config-if)# ppp multilink group 1
Router(config)# interface serial 0/0/0
Router(config-if)# ppp multilink group 1
Router(config)# interface serial 0/0/1
Router(config-if)# ppp multilink group 1
💡 CHAP vs PAP: CHAP is more secure because it never sends the password across the link. Use CHAP whenever possible.

🚇 GRE Tunnels — Generic Routing Encapsulation

GRE is a tunneling protocol that encapsulates any network layer protocol inside IP. Used for VPNs, routing protocols over internet, and connecting discontinuous networks.

┌─────────────┐ Internet ┌─────────────┐ │ HQ Router │══════════════════════════│ Branch Router│ │ 10.0.0.1 │ GRE Tunnel │ 10.0.0.2 │ │ Tunnel IP: │══════════════════════════│ Tunnel IP: │ │ 192.168.0.1 │ │ 192.168.0.2 │ └─────────────┘ └─────────────┘

GRE Configuration (Cisco):

! HQ Router (10.0.0.1 - physical interface)
Router(config)# interface tunnel 0
Router(config-if)# ip address 192.168.0.1 255.255.255.252
Router(config-if)# tunnel source 10.0.0.1
Router(config-if)# tunnel destination 10.0.0.2
Router(config-if)# exit

! Branch Router (10.0.0.2 - physical interface)
Router(config)# interface tunnel 0
Router(config-if)# ip address 192.168.0.2 255.255.255.252
Router(config-if)# tunnel source 10.0.0.2
Router(config-if)# tunnel destination 10.0.0.1

Routing over GRE (OSPF over Tunnel):

Router(config)# router ospf 1
Router(config-router)# network 192.168.0.0 0.0.0.3 area 0
💡 GRE + IPsec: GRE tunnels are not encrypted. For secure VPNs, combine GRE with IPsec transport mode.

🖧 Frame Relay — Legacy WAN (Still on Exams!)

Frame Relay is a packet-switched WAN technology that was widely used from 1990s-2010s. Still appears on CCNA exams despite being legacy.

Frame Relay Concepts:

  • DLCI (Data Link Connection Identifier): Local identifier (16-1007)
  • LMI (Local Management Interface): Signaling protocol between router and Frame Relay switch
  • PVC (Permanent Virtual Circuit): Predefined path through the Frame Relay cloud
  • DE (Discard Eligibility): Bit indicating packets that can be dropped during congestion

Frame Relay Configuration (Cisco):

! Interface configuration
Router(config)# interface serial 0/0/0
Router(config-if)# encapsulation frame-relay
Router(config-if)# frame-relay lmi-type ansi
Router(config-if)# ip address 10.0.0.1 255.255.255.0

! Static mapping (for non-broadcast environments)
Router(config-if)# frame-relay map ip 10.0.0.2 102 broadcast
⚠️ Exam Note: Frame Relay is obsolete in production but appears on CCNA/CCNP exams. Focus on understanding DLCI, LMI, and Inverse ARP.

🏷️ MPLS — Multi-Protocol Label Switching

MPLS is the modern WAN backbone technology used by ISPs and large enterprises. Instead of routing based on IP destination, it uses labels for fast forwarding.

┌─────────┐ Label ┌─────────┐ Label ┌─────────┐ │ Router │═════════════│ Router │═════════════│ Router │ │ (PE) │ (LSR) │ (P) │ (LSR) │ (PE) │ └─────────┘ └─────────┘ └─────────┘ │ │ │ Customer A MPLS Cloud Customer A (VPN) (ISP Core) (VPN)

MPLS Advantages:

  • ✅ Fast forwarding — Labels, not IP lookups
  • ✅ Traffic engineering — Control which path traffic takes
  • ✅ Layer 2/Layer 3 VPNs — MPLS VPNs are industry standard
  • ✅ QoS integration — Labels carry priority information

MPLS Terms:

  • PE (Provider Edge): Router connecting customer to MPLS network
  • CE (Customer Edge): Customer router connecting to PE
  • P (Provider): Core router that forwards based on labels
  • LSR (Label Switch Router): Any router that forwards MPLS labels
  • LDP (Label Distribution Protocol): Distributes labels between routers
💡 MPLS vs Traditional Routing: Traditional routing performs IP lookup at every hop. MPLS does one IP lookup at the edge, then uses fast label switching in the core.

📝 Practice Questions

Q1: Why is HDLC not recommended for multi-vendor environments?
Answer: Cisco HDLC is proprietary. It uses a different frame format than standard HDLC, so it won't communicate with non-Cisco routers.
Q2: What is the main advantage of CHAP over PAP in PPP?
Answer: CHAP never sends the password across the link. It uses a challenge-response mechanism with hashed values, making it much more secure than PAP.
Q3: Why would you use a GRE tunnel?
Answer: GRE can encapsulate any network layer protocol (not just IP). It's used to route protocols like OSPF over the internet or to connect discontinuous networks.
Q4: What does DLCI represent in Frame Relay?
Answer: DLCI (Data Link Connection Identifier) is a locally significant identifier for PVCs. It's like a virtual circuit number.
Q5: How does MPLS achieve faster forwarding than traditional IP routing?
Answer: Traditional IP routing requires a longest-prefix match lookup at each hop. MPLS uses fixed-length labels that can be looked up quickly, often in hardware (ASIC).

🎓 Key Takeaways (WAN Technologies)

  • ✅ HDLC: Cisco default, proprietary, no authentication — use for Cisco-only links
  • ✅ PPP: Industry standard, supports CHAP/PAP authentication, multi-vendor compatible
  • ✅ GRE tunnels: Encapsulate any protocol, combine with IPsec for security
  • ✅ Frame Relay: Legacy technology (DLCI, LMI, PVC) — still on exams
  • ✅ MPLS: Modern ISP backbone, label switching, VPN support, traffic engineering
  • ✅ PPP Multilink: Combine multiple physical links into one logical link
  • ✅ CHAP > PAP: Always choose CHAP for better security
