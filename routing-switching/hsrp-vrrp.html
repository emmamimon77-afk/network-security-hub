🔄 HSRP/VRRP — First Hop Redundancy Protocols

Hot Standby Router Protocol (HSRP) and Virtual Router Redundancy Protocol (VRRP) provide gateway redundancy. If your default gateway router fails, another router automatically takes over — clients don't even notice.

💡 Prerequisite: Basic routing and IP addressing. HSRP/VRRP are essential for high-availability networks.

🎯 The Problem: Single Point of Failure

Without First Hop Redundancy: Client (192.168.1.10) │ ▼ Router A (192.168.1.1) ← DEFAULT GATEWAY │ ▼ INTERNET 🔴 If Router A fails → Client has NO gateway → NO internet
💡 The solution: Two routers share a virtual IP address. One is active (forwards traffic), the other is standby (ready to take over).

📊 HSRP vs VRRP vs GLBP

FeatureHSRPVRRPGLBP
Vendor Cisco proprietary Open standard (RFC 5798) Cisco proprietary
Virtual IP Yes (separate from physical IPs) Yes (separate from physical IPs) Yes (multiple virtual IPs possible) Active/Standby 1 Active, 1+ Standby 1 Master, 1+ Backup Multiple Active (load balancing) Load balancing ❌ No (only one active router) ❌ No (only one master router) ✅ Yes (up to 4 routers per group) Authentication MD5, text MD5, text (RFC) MD5, text
💡 Which to use? HSRP for Cisco-only networks, VRRP for multi-vendor, GLBP for load balancing.

🔵 HSRP — Hot Standby Router Protocol (Cisco)

How HSRP Works

Virtual IP: 192.168.1.254 (clients use this) Virtual MAC: 0000.0c07.ac01 ┌─────────────────────────────────────┐ │ Client PC (192.168.1.10) │ │ Default Gateway: 192.168.1.254 │ └─────────────────────────────────────┘ │ ┌──────────────────┴──────────────────┐ │ │ ▼ ▼ ┌───────────────────┐ ┌───────────────────┐ │ Router A │ │ Router B │ │ ACTIVE (Priority 110) │ │ STANDBY (Priority 100)│ │ Physical IP: │ │ Physical IP: │ │ 192.168.1.1 │ │ 192.168.1.2 │ └───────────────────┘ └───────────────────┘ │ │ └──────────────────┬──────────────────┘ │ ▼ INTERNET

HSRP States

  • Initial: HSRP not yet started
  • Learn: Router looking for active router
  • Listen: Router knows active/standby, not participating
  • Speak: Router sends hello messages
  • Standby: Ready to take over (sends hellos)
  • Active: Forwards traffic for virtual IP

HSRP Configuration (Cisco)

! Router A (ACTIVE - higher priority)
RouterA(config)# interface gigabitEthernet 0/0
RouterA(config-if)# ip address 192.168.1.1 255.255.255.0
RouterA(config-if)# standby 1 ip 192.168.1.254
RouterA(config-if)# standby 1 priority 110
RouterA(config-if)# standby 1 preempt
RouterA(config-if)# standby 1 authentication md5 key-string SecurePass
RouterA(config-if)# standby 1 timers 3 10

! Router B (STANDBY - lower priority)
RouterB(config)# interface gigabitEthernet 0/0
RouterB(config-if)# ip address 192.168.1.2 255.255.255.0
RouterB(config-if)# standby 1 ip 192.168.1.254
RouterB(config-if)# standby 1 priority 100
RouterB(config-if)# standby 1 authentication md5 key-string SecurePass

! Verification commands
Router# show standby
Router# show standby brief
📋 Understanding the commands:
  • standby 1 ip ... — Group 1, virtual IP address (clients use this)
  • standby priority — Higher priority = becomes active (default 100)
  • standby preempt — If active router fails and returns, it retakes active role
  • standby authentication — Prevents rogue routers from joining
  • standby timers 3 10 — Hello every 3 sec, hold timer 10 sec (fast failover)
RouterA# show standby brief
P indicates configured to preempt.

🟢 VRRP — Virtual Router Redundancy Protocol (Open Standard)

VRRP Differences from HSRP

  • ✅ Open standard — Works with non-Cisco routers (Juniper, Huawei, etc.)
  • ✅ VRRP uses "Master/Backup" instead of Active/Standby
  • ✅ Virtual IP can be one of the physical IPs (not required to be separate)
  • ✅ IPv6 support built-in (HSRPv6 also exists)

VRRP Configuration (Cisco)

! Router A (MASTER - higher priority)
RouterA(config)# interface gigabitEthernet 0/0
RouterA(config-if)# ip address 192.168.1.1 255.255.255.0
RouterA(config-if)# vrrp 1 ip 192.168.1.254
RouterA(config-if)# vrrp 1 priority 110
RouterA(config-if)# vrrp 1 preempt
RouterA(config-if)# vrrp 1 authentication md5 key-string SecurePass

! Router B (BACKUP - lower priority)
RouterB(config)# interface gigabitEthernet 0/0
RouterB(config-if)# ip address 192.168.1.2 255.255.255.0
RouterB(config-if)# vrrp 1 ip 192.168.1.254
RouterB(config-if)# vrrp 1 priority 100

! Verification
Router# show vrrp
Router# show vrrp brief

📊 HSRP, VRRP, GLBP — When to Use Each

Use CaseRecommended ProtocolReason
Pure Cisco network, active/standby only HSRP Cisco proprietary, simple, widely deployed
Multi-vendor environment (Cisco + Juniper + Huawei) VRRP Open standard, interoperable
Need load balancing (utilize both routers) GLBP Up to 4 active routers sharing traffic

🔧 Troubleshooting First Hop Redundancy

Common Issues:

  • No active router elected — Check that HSRP/VRRP is configured on both routers
  • Both routers are active — Authentication mismatch or different group numbers
  • Failover not working — Check preempt configuration
  • Clients can't ping virtual IP — Verify interface status and routing

⚡ HSRP on Layer 3 Switches (Special Notes)

HSRP works almost identically on L3 switches, but there's one critical requirement:

! Required on L3 switches for each VLAN
Router(config)# interface vlan 10
Router(config-if)# standby 1 ip 192.168.1.254
Router(config-if)# standby 1 priority 110
Router(config-if)# standby 1 preempt

! Enable IP routing (must be enabled!)
Router(config)# ip routing

📝 Practice Questions

Q1: What is the purpose of HSRP/VRRP?
Answer: To provide gateway redundancy — if the active router fails, a standby router takes over so clients don't lose connectivity.
Q2: What is the main difference between HSRP and VRRP?
Answer: HSRP is Cisco proprietary; VRRP is an open standard (RFC 5798) that works with any vendor.
Q3: What command makes a router reclaim active role after it recovers from a failure?
Answer: standby preempt (HSRP) or vrrp preempt (VRRP)
Q4: Which protocol supports load balancing across multiple routers?
Answer: GLBP (Gateway Load Balancing Protocol) — up to 4 active routers can share traffic.
Q5: What is the default HSRP hello and hold timer?
Answer: Hello timer: 3 seconds, Hold timer: 10 seconds.

🎓 Key Takeaways (HSRP/VRRP)

  • ✅ HSRP — Cisco proprietary, Active/Standby, virtual IP + virtual MAC
  • ✅ VRRP — Open standard, Master/Backup, works with all vendors
  • ✅ GLBP — Cisco only, load balancing across up to 4 routers
  • ✅ Higher priority = Active/Master (default priority 100)
  • ✅ Preempt — Allows active router to reclaim role after recovery
  • ✅ Authentication — Prevents rogue routers from joining the group
  • ✅ Virtual IP must match on all routers in the same group
