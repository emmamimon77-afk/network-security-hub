⚡ EIGRP — Complete Guide

Enhanced Interior Gateway Routing Protocol (EIGRP) = Cisco's advanced distance-vector routing protocol. Combines the best of distance-vector and link-state protocols. Fast convergence, easy configuration, and Cisco proprietary.

🎯 Why EIGRP Matters

  • ✅ Very fast convergence — Often sub-second (faster than OSPF)
  • ✅ Easy to configure — Simpler than OSPF (no areas, fewer commands)
  • ✅ Efficient bandwidth usage — Only sends updates when topology changes
  • ✅ Load balancing — Supports unequal-cost load balancing (unique to EIGRP!)
  • ✅ Widely used — Still common in Cisco-only enterprise networks
💡 Real-world: Many large enterprises run EIGRP internally because it's easier to manage and troubleshoot than OSPF — but only if they're 100% Cisco.
⚠️ Important: EIGRP is Cisco proprietary. For multi-vendor networks (Cisco + Juniper + Arista), use OSPF.

📚 Core EIGRP Concepts (Must Know)

1. Autonomous System (AS) Number

EIGRP routers must share the same AS number to become neighbors.

router eigrp 100 ← AS number (1-65535)

2. Metric (Composite)

EIGRP uses a composite metric — bandwidth and delay by default:

Metric = (Bandwidth + Delay) × 256
  • Bandwidth: Slowest link path (in kilobits). Lower bandwidth = higher metric.
  • Delay: Cumulative delay (in microseconds). Higher delay = higher metric.
  • K values: You can also include load, reliability, MTU (but rarely used).

3. Successor & Feasible Successor (Dual Algorithm)

EIGRP's loop prevention mechanism.

  • Successor: The best route to a destination (installed in routing table).
  • Feasible successor: A backup route (lower metric than successor's reported distance). Installed in topology table, ready to use if successor fails.
Example: RouterA wants to reach 192.168.1.0/24. Path 1: RouterA → RouterB (metric 1000) ← SUCCESSOR Path 2: RouterA → RouterC → RouterB (metric 1500) ← FEASIBLE SUCCESSOR (if 1500 < 1000 × K?) If Path 1 fails, Path 2 takes over immediately (no recomputation needed)!

4. Hello Protocol

  • Hello interval: 5 seconds (LAN) / 60 seconds (WAN)
  • Hold time: 15 seconds (LAN) / 180 seconds (WAN)

⚙️ EIGRP Configuration (Cisco CLI)

Basic EIGRP Configuration

Router(config)# router eigrp 100
Router(config-router)# network 192.168.1.0
Router(config-router)# network 10.0.0.0
Router(config-router)# no auto-summary
Router(config-router)# passive-interface gigabitEthernet 0/0
Router(config-router)# end
💡 Why `no auto-summary`? EIGRP used to automatically summarize networks at classful boundaries (A/B/C). This is almost always undesirable. Always use no auto-summary!

EIGRP Network Statement (Classless)

Unlike OSPF, EIGRP uses classful network statements by default — but with wildcard masks, you can be precise.

network 192.168.1.0 0.0.0.255 ← Enable EIGRP on 192.168.1.0/24
network 10.0.0.0 ← Enable EIGRP on ALL 10.x.x.x interfaces!

Verification Commands

Router# show ip eigrp neighbors
Router# show ip eigrp topology
Router# show ip route eigrp
Router# show ip protocols

Configuration Example Output

⚖️ Unequal-Cost Load Balancing (The EIGRP Superpower)

OSPF only load-balances across equal-cost paths. EIGRP can use unequal-cost paths with the variance command.

RouterA to 192.168.1.0/24: Path 1: Metric 1000 (Gigabit) ← Successor Path 2: Metric 1500 (FastEthernet) ← Feasible Successor variance 2 means: Install routes with metric ≤ 1000 × 2 = 2000. Path 2 (1500 < 2000) is installed! Traffic splits (70% / 30%).
Router(config)# router eigrp 100
Router(config-router)# variance 2
Router(config-router)# end

Router# show ip route
D 192.168.2.0/24 [90/1000] via 10.0.0.2, 00:00:01, Serial0/0/0
[90/1500] via 10.0.0.6, 00:00:01, Serial0/0/1
💡 Use case: You have a 100 Mbps link (cost 10) and a 50 Mbps backup (cost 20). Variance 2 lets you use both for load balancing!

🛡️ EIGRP Security (MD5 Authentication)

Router(config)# key chain EIGRP-KEYS
Router(config-keychain)# key 1
Router(config-keychain-key)# key-string SecurePass123
Router(config-keychain-key)# exit
Router(config)# interface serial 0/0/0
Router(config-if)# ip authentication mode eigrp 100 md5
Router(config-if)# ip authentication key-chain eigrp 100 EIGRP-KEYS
⚠️ Always authenticate EIGRP! Without authentication, attackers can inject fake EIGRP routes and redirect traffic.

🔧 Troubleshooting EIGRP

❌ Neighbors not forming?
Check:
1. Same AS number on both routers?
2. Same K values? (show ip protocols)
3. No ACL blocking multicast 224.0.0.10?
4. Authentication mismatch?
❌ Routes not in routing table?
Check:
1. Are routes in topology table? show ip eigrp topology
2. Is no auto-summary configured?
3. Passive-interface preventing Hellos?
❌ Unequal-cost load balancing not working?
Check:
1. Is variance configured? show ip eigrp topology
2. Are feasible successors available?
3. Try variance 2, then variance 4

📝 Practice Questions (CCNA-Style)

Q1: What command enables EIGRP with AS number 200?

Answer: router eigrp 200 in global configuration mode.

Q2: What is the default Hello interval on an Ethernet (LAN) interface for EIGRP?

Answer: 5 seconds.

Q3: What command allows EIGRP to perform unequal-cost load balancing?

Answer: variance [multiplier] within the EIGRP router configuration.

Q4: Why is no auto-summary important in EIGRP?

Answer: It prevents EIGRP from automatically summarizing networks at classful boundaries (A/B/C), which can cause routing issues in modern VLSM networks.

Q5: What is the difference between a Successor and a Feasible Successor?

Answer: Successor = best route (installed in routing table). Feasible Successor = backup route (in topology table, ready to use if successor fails).

📝 EIGRP vs OSPF — Quick Decision Guide

Choose EIGRP when:
• Your network is 100% Cisco
• You need unequal-cost load balancing
• You prefer simpler configuration (no areas)
• You want very fast convergence (sub-second)

Choose OSPF when:
• You have multiple vendors (Juniper, Arista, Huawei)
• You need an open standard (RFC)
• You have extremely large networks (thousands of routers)
• You want to avoid vendor lock-in

🎓 Key Takeaways (EIGRP Mastery)

  • ✅ EIGRP = Cisco proprietary — only works between Cisco devices
  • ✅ AS number must match for routers to become neighbors
  • ✅ DUAL algorithm = Successor (best) + Feasible Successor (backup)
  • ✅ Metric = Bandwidth + Delay — lower is better
  • ✅ Variance enables unequal-cost load balancing (unique to EIGRP)
  • ✅ Always use `no auto-summary` — prevents classful summarization
  • ✅ EIGRP converges faster than OSPF (sub-second vs seconds)
