🗺️ Static Routing — Complete Guide

Static Routing = Manually configured routes that tell a router exactly where to send traffic. No dynamic discovery, no protocols — just explicit instructions.

🎯 Why Static Routing Matters

  • ✅ Security — No routing updates to intercept or spoof
  • ✅ Zero overhead — No bandwidth wasted on routing protocols
  • ✅ Predictable — Routes never change unless you change them
  • ✅ Simplicity — Perfect for small networks (1-5 routers)
  • ✅ Foundation skill — Dynamic routing builds on static concepts
💡 Real-world: Stub networks (branch offices, home routers) use a default static route. Edge routers use static routes to ISP.

📐 Static Route Anatomy (The Formula)

Cisco Syntax: ip route [destination_network] [subnet_mask] [next_hop_ip | exit_interface] [administrative_distance]

Example 1 — Next-hop IP:
ip route 192.168.2.0 255.255.255.0 10.0.0.2
"To reach 192.168.2.0/24, send packets to router at 10.0.0.2."
Example 2 — Exit interface:
ip route 10.1.1.0 255.255.255.0 serial0/0/0
"To reach 10.1.1.0/24, send packets out serial0/0/0."
Example 3 — Default route (gateway of last resort):
ip route 0.0.0.0 0.0.0.0 203.0.113.1
"If no specific route matches, send to 203.0.113.1 (ISP gateway)."
📌 L3 Switch Users: Before adding static routes, ensure ip routing is enabled and SVIs (interface vlan X) or no switchport ports are used. Same ip route commands work!

Branch2 needs to reach HQ (192.168.1.0/24):
Route: ip route 192.168.1.0 255.255.255.0 10.0.0.6
(Send to Branch1, which knows how to reach HQ)

Each router must know how to reach ALL non-local networks!

🏆 Administrative Distance (Which Route Wins?)

Administrative Distance (AD) = Trustworthiness of route source. Lower = preferred.

Route SourceADWhen Used
Connected interface0Router's own IP networks — always highest priority
Static route1Manually configured — very trusted
EIGRP (internal)90Cisco's IGP
OSPF110Open standard IGP
RIP120Old distance vector protocol
External EIGRP170Routes redistributed into EIGRP
Unknown / Invalid255Never used
Modifying AD (Backup Route):
ip route 192.168.2.0 255.255.255.0 10.0.0.2 ← AD 1 (primary)
ip route 192.168.2.0 255.255.255.0 10.0.0.5 2 ← AD 2 (backup)
Primary route AD 1 wins. If failed, AD 2 takes over!

🔄 Floating Static Routes (Backup Links)

When you have two paths to a destination, use a higher AD for the backup path.

Primary link (fiber): RouterA ──10.0.0.0/30── RouterB (AD 1) Backup link (cellular): RouterA ──10.0.0.4/30── RouterB (AD 100)
Primary (preferred):
ip route 192.168.2.0 255.255.255.0 10.0.0.2

Backup (only if primary down):
ip route 192.168.2.0 255.255.255.0 10.0.0.6 100
💡 Use case: Branch office with fiber + 4G LTE backup. Fiber is primary (AD 1), cellular is backup (AD 100).

🎯 Routing Table Lookup (How Router Decides)

Algorithm: Longest prefix match wins.

Routing table entries:
S 192.168.0.0/16 [1/0] via 10.0.0.2
S 192.168.5.0/24 [1/0] via 10.0.0.4
S* 0.0.0.0/0 [1/0] via 203.0.113.1

Destination 192.168.5.1: Matches /24 (24 bits) → 10.0.0.4
Destination 192.168.9.1: Matches /16 (16 bits) → 10.0.0.2
Destination 8.8.8.8: Only matches default route → 203.0.113.1

⚠️ Common mistake: Adding a static route with 0.0.0.0 0.0.0.0 (default) does NOT override existing specific routes. Specific always wins!

🛡️ Static Routing Security

  • ✅ No route advertisements — Attackers can't inject fake OSPF/EIGRP updates
  • ✅ No route calculations — Less CPU overhead = more resistant to DOS
  • ⚠️ Risk: If route is wrong, traffic loops until manually fixed
  • ⚠️ Risk: No automatic failover (unless floating static configured)
Best Practice: Use static routes only for:
- Stub networks (single exit point)
- Default routes to ISP
- Backup for dynamic routing
- Very small networks (≤5 routers)

📝 Practice Questions

Q1: What command adds a static route to 10.1.1.0/24 via next-hop 172.16.1.1?

Answer: ip route 10.1.1.0 255.255.255.0 172.16.1.1

Q2: Your router has two static routes to 192.168.5.0/24 — one AD 1, one AD 100. Which is used?

Answer: AD 1 (lower AD wins). AD 100 is backup only.

Q3: What does ip route 0.0.0.0 0.0.0.0 203.0.113.1 do?

Answer: Default route — sends all traffic without a specific match to 203.0.113.1 (gateway of last resort).

Q4: In show ip route, what does "S*" indicate?

Answer: Default static route (0.0.0.0/0).

Q5: Two static routes exist: /16 and /24 to same destination. Which wins?

Answer: /24 (longest prefix match).

Q6: Why would you use a floating static route?

Answer: For automatic backup when primary route fails (higher AD = backup).

🧪 Lab Exercise: Build a 3-Router Static Route Network

[PC1] [PC2] [PC3] │ │ │ VLAN10 VLAN20 VLAN30 │ │ │ [R1]──────[R2]──────[R3] │ │ │ 10.0.0.0/30 10.0.0.4/30 10.0.0.8/30 │ │ │ LAN: LAN: LAN: 192.168.1.0/24 192.168.2.0/24 192.168.3.0/24

Step 1: Configure Interfaces on Each Router

Router1 (R1):
interface gig0/0
ip address 192.168.1.1 255.255.255.0
no shutdown
interface serial0/0/0
ip address 10.0.0.1 255.255.255.252
no shutdown
Router2 (R2):
interface gig0/0
ip address 192.168.2.1 255.255.255.0
no shutdown
interface serial0/0/0
ip address 10.0.0.2 255.255.255.252
no shutdown
interface serial0/0/1
ip address 10.0.0.5 255.255.255.252
no shutdown
Router3 (R3):
interface gig0/0
ip address 192.168.3.1 255.255.255.0
no shutdown
interface serial0/0/0
ip address 10.0.0.6 255.255.255.252
no shutdown

Step 2: Add Static Routes

R1 needs routes to R2's LAN and R3's LAN:
ip route 192.168.2.0 255.255.255.0 10.0.0.2
ip route 192.168.3.0 255.255.255.0 10.0.0.2

R2 needs routes to both R1 and R3 LANs:
ip route 192.168.1.0 255.255.255.0 10.0.0.1
ip route 192.168.3.0 255.255.255.0 10.0.0.6

R3 needs routes to R1 and R2 LANs:
ip route 192.168.1.0 255.255.255.0 10.0.0.5
ip route 192.168.2.0 255.255.255.0 10.0.0.5

Step 3: Verify Connectivity

R1# show ip route
R1# ping 192.168.2.1
R1# ping 192.168.3.1
R1# traceroute 192.168.3.1
💡 Challenge: Add a backup link between R1 and R3 directly (10.0.0.8/30). Create floating static routes with AD 100 for failover.

🎓 Key Takeaways (Static Routing Mastery)

  • ✅ Syntax: ip route [dest] [mask] [next-hop | interface] [AD]
  • ✅ Connected routes (C/L) appear automatically — no config needed
  • ✅ Default route (S*) = ip route 0.0.0.0 0.0.0.0 [gateway]
  • ✅ Lower AD = preferred (static default AD 1)
  • ✅ Floating static = Higher AD for backup links
  • ✅ Longest prefix match wins in routing table lookup
  • ✅ Static routes are NOT automatic — both ends need routes
