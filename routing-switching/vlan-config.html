🏷️ VLAN Configuration — Complete Guide

Virtual Local Area Networks (VLANs) = Divide one physical switch into multiple isolated broadcast domains. Essential for security, performance, and network design.

🎯 Why VLANs Matter

  • ✅ Security — Isolate sensitive departments (HR, Finance) from guest Wi-Fi
  • ✅ Performance — Reduce broadcast traffic (ARP, DHCP)
  • ✅ Cost savings — One switch does the work of many
  • ✅ Flexibility — Move devices without re-cabling
💡 Real-world: A university uses VLANs to separate Students (VLAN 10), Faculty (VLAN 20), and IT (VLAN 99). No student can reach faculty servers.

📊 VLAN Types (Standard Ranges)

VLAN RangeTypeUsage
1DefaultAll ports start here. Security Risk — avoid using
2-1001NormalStandard user VLANs (can be created/deleted)
1002-1005ReservedToken Ring / FDDI — not used in Ethernet
1006-4094ExtendedNeed VTP Transparent mode to use

Special VLANs to Know

  • Native VLAN (default 1): Untagged traffic on trunk ports — Change this!
  • Management VLAN: For SSH/Telnet access to switch
  • Voice VLAN: QoS priority for IP phones
  • Blackhole VLAN: Unused ports assigned here for security

⚙️ VLAN Configuration (Cisco CLI)

Creating a VLAN

Switch> enable
Switch# configure terminal
Switch(config)# vlan 10
Switch(config-vlan)# name Engineering
Switch(config-vlan)# vlan 20
Switch(config-vlan)# name Marketing
Switch(config-vlan)# end

Assigning Ports to VLAN (Access Mode)

Switch(config)# interface range gigabitEthernet 0/1-12
Switch(config-if-range)# switchport mode access
Switch(config-if-range)# switchport access vlan 10
Switch(config-if-range)# no shutdown
Switch(config-if-range)# exit

Switch(config)# interface range gigabitEthernet 0/13-24
Switch(config-if-range)# switchport mode access
Switch(config-if-range)# switchport access vlan 20

Verifying VLAN Configuration

Switch# show vlan brief
Switch# show interfaces status
Switch# show interfaces trunk

Trunk Configuration (Connect Switches)

Switch1(config)# interface gigabitEthernet 0/24
Switch1(config-if)# switchport mode trunk
Switch1(config-if)# switchport trunk native vlan 999 # Change native VLAN for security
Switch1(config-if)# switchport trunk allowed vlan 10,20,30
Switch1(config-if)# no shutdown

💡 Modern networks use 802.1Q exclusively. ISL is obsolete.

🌐 Inter-VLAN Routing (Making VLANs Talk)

VLANs are isolated by default. To communicate, you need a router.

Option 1: Router-on-a-Stick (Single Router Interface)

Router(config)# interface gigabitEthernet 0/0
Router(config-if)# no shutdown
Router(config)# interface gigabitEthernet 0/0.10
Router(config-subif)# encapsulation dot1Q 10
Router(config-subif)# ip address 192.168.10.1 255.255.255.0
Router(config-subif)# exit
Router(config)# interface gigabitEthernet 0/0.20
Router(config-subif)# encapsulation dot1Q 20
Router(config-subif)# ip address 192.168.20.1 255.255.255.0

Option 2: Layer 3 Switch (SVI - Switch Virtual Interface)

Switch(config)# ip routing
Switch(config)# interface vlan 10
Switch(config-if)# ip address 192.168.10.1 255.255.255.0
Switch(config-if)# no shutdown
Switch(config)# interface vlan 20
Switch(config-if)# ip address 192.168.20.1 255.255.255.0
Switch(config-if)# no shutdown

🛡️ VLAN Security (Critical!)

🔴 Threat 1: VLAN Hopping

Attacker tricks switch into sending packets to another VLAN.

  • Switch Spoofing: Attacker pretends to be a switch using DTP
  • Double Tagging: Attacker adds two VLAN tags
Defenses:
  • ✅ Disable DTP: switchport nonegotiate on all access ports
  • ✅ Set unused ports to switchport mode access + assign to blackhole VLAN
  • ✅ Change native VLAN to unused number (e.g., 999)
  • ✅ switchport trunk native vlan 999

🔴 Threat 2: ARP Spoofing / MAC Flooding

Defenses:
  • ✅ DAI (Dynamic ARP Inspection): ip arp inspection vlan 10,20
  • ✅ Port Security: switchport port-security maximum 2
  • ✅ DHCP Snooping: ip dhcp snooping vlan 10

🔴 Threat 3: Native VLAN Mismatch

Defense: Explicitly set same native VLAN on both ends:
switchport trunk native vlan 999

🟢 Security Best Practices Checklist

  • ✅ Never use VLAN 1 for user traffic
  • ✅ Change native VLAN from 1 to unused number
  • ✅ Disable DTP on all access ports
  • ✅ Shut down unused ports and assign to blackhole VLAN
  • ✅ Enable port security with MAC limits
  • ✅ Use DHCP snooping + DAI on user VLANs
  • ✅ Encrypt management traffic (SSH not Telnet)

📡 VTP (VLAN Trunking Protocol) — Cisco Proprietary

VTP synchronizes VLAN databases across switches.

⚠️ VTP Warning: A server with higher revision number can wipe out entire VLAN database! Use Transparent mode for security.
Switch(config)# vtp mode transparent
Switch(config)# vtp domain MyCompany
Switch(config)# vtp password SecurePass123

🔒 Private VLANs (PVLANs) — Ultra Isolation

Prevents devices in same VLAN from talking to each other. Used for DMZ, guest Wi-Fi, multi-tenant environments.

Switch(config)# vlan 100
Switch(config-vlan)# private-vlan primary
Switch(config-vlan)# vlan 101
Switch(config-vlan)# private-vlan isolated
Switch(config)# interface gigabitEthernet 0/1
Switch(config-if)# switchport mode private-vlan host
Switch(config-if)# switchport private-vlan host-association 100 101
💡 Use case: Hotel guest network — guests cannot see each other's devices, only the gateway.

📝 Practice Questions

Q1: What command creates VLAN 30 named "Sales"?

Answer: vlan 30 → name Sales in global config mode.

Q2: Two switches connected via trunk. What must match on both ends?

Answer: Native VLAN, allowed VLAN list, trunk encapsulation (802.1Q).

Q3: How do you prevent VLAN hopping attacks?

Answer: Disable DTP, change native VLAN, use allowed VLAN list on trunks, shut unused ports.

Q4: What's the difference between access and trunk ports?

Answer: Access = single VLAN (end devices). Trunk = multiple VLANs (switch-to-switch).

Q5: Why is VLAN 1 dangerous?

Answer: It's the default native VLAN, carries untagged traffic, and is often used for management — creating a security risk.

🎓 Key Takeaways (VLAN Mastery)

  • ✅ VLANs = Security + Performance — Isolate broadcast domains
  • ✅ 802.1Q is the standard — Avoid ISL (obsolete)
  • ✅ Never use VLAN 1 for user traffic — Change native VLAN!
  • ✅ Inter-VLAN routing needs router (ROAS) or L3 switch (SVI)
  • ✅ VTP Transparent is safer than Server/Client
  • ✅ Private VLANs prevent host-to-host communication
  • ✅ Security checklist: disable DTP, port security, DHCP snooping, DAI
