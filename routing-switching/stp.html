🌲 Spanning Tree Protocol (STP) — Complete Guide

STP prevents network loops in Ethernet networks. Without STP, a single broadcast storm can bring down your entire network. This is essential for any network with redundant links.

💡 Prerequisite: Basic switching concepts (VLANs, MAC addresses). STP operates at Layer 2.

🎯 Why STP Matters

  • ✅ Prevents broadcast storms — One loop can flood the network
  • ✅ Enables redundancy — Multiple paths without breaking the network
  • ✅ Automatic failover — When a link fails, STP recalculates
  • ✅ Every switch runs it — STP is enabled by default on almost all switches
⚠️ Without STP: A single loop can cause broadcast frames to circulate forever, consuming all bandwidth and CPU — network goes down in seconds.

🔁 The Problem: What Happens Without STP?

┌─────────┐ ┌─────────┐ │ Switch1 │═════════│ Switch2 │ └────┬────┘ └────┬────┘ │ │ │ ┌─────────┐ │ └══════│ Switch3 │══┘ └─────────┘ This topology has a loop. Without STP: 1. Broadcast frame enters Switch1 2. Forwarded to Switch2 AND Switch3 3. Switch2 sends to Switch3, Switch3 sends to Switch2 4. Frames circulate FOREVER → BROADCAST STORM
💡 Real-world analogy: Two people shouting in a circular room — the echo never stops.

⚙️ How STP Works (The 4-Step Process)

Step 1: Elect a Root Bridge

The switch with the lowest Bridge ID (Priority + MAC address) becomes the root.

Bridge ID = Priority (2 bytes) + MAC Address (6 bytes)
Default priority: 32768
Lower priority = more likely to become root

Step 2: Determine Root Ports

Each non-root switch selects one port with the lowest cost to reach the root.

Step 3: Determine Designated Ports

On each segment, one port is elected as the designated port (forwarding).

Step 4: Block Alternate Ports

All other ports become alternate/backup ports and are placed in blocking state.

🔄 STP Port States (5 States)

Listening No No 15 sec Electing root and designated ports

💡 Total convergence time (default): Blocking → Forwarding takes ~50 seconds (20 + 15 + 15). RSTP reduces this to < 6 seconds.

⚙️ STP Configuration (Cisco CLI)

Configuring Root Bridge

! Set switch as root for VLAN 1
Switch(config)# spanning-tree vlan 1 root primary

! Or manually set priority (lower = more likely to be root)
Switch(config)# spanning-tree vlan 1 priority 4096

Configuring Port Cost (Influences Path Selection)

! Set port cost on specific interface
Switch(config)# interface gigabitEthernet 0/12
Switch(config-if)# spanning-tree cost 4

! Verify STP status
Switch# show spanning-tree
Switch# show spanning-tree interface gig0/12

PortFast (for End Devices)

! Bypasses listening/learning states — port goes directly to forwarding
Switch(config)# interface gigabitEthernet 0/1
Switch(config-if)# spanning-tree portfast

! Enable PortFast on all access ports
Switch(config)# spanning-tree portfast default

BPDUGuard (Security)

! Disable port if BPDU is received (prevents unauthorized switches)
Switch(config)# interface gigabitEthernet 0/1
Switch(config-if)# spanning-tree bpduguard enable

⚡ RSTP (Rapid Spanning Tree Protocol) — 802.1w

RSTP is the modern replacement for STP, with much faster convergence (< 6 seconds vs 50 seconds).

FeatureSTP (802.1D)RSTP (802.1w)
Convergence time ~50 seconds < 6 seconds
Port states Blocking, Listening, Learning, Forwarding Discarding, Learning, Forwarding
Backward compatible — ✅ Yes (works with legacy STP)
! Enable RSTP (default on most modern switches)
Switch(config)# spanning-tree mode rapid-pvst

🌳 MST (Multiple Spanning Tree) — 802.1s

MST allows multiple VLANs to be mapped to a single spanning tree instance, reducing CPU load.

! Configure MST region
Switch(config)# spanning-tree mst configuration
Switch(config-mst)# name REGION1
Switch(config-mst)# revision 1
Switch(config-mst)# instance 1 vlan 10,20,30
Switch(config-mst)# instance 2 vlan 40,50,60
Switch(config-mst)# end

🔧 STP Troubleshooting

Common STP Issues:

  • Root bridge election problems — Check bridge priorities: show spanning-tree
  • No redundant path — A port in blocking state is normal (that's the backup!)
  • Slow network after link failure — Classic STP takes 50 seconds; migrate to RSTP
  • PortFast not working — Port may still go through listening/learning
  • BPDUGuard triggered — Someone plugged in a switch on an access port
📋 Useful Commands:
show spanning-tree
show spanning-tree root
show spanning-tree interface gig0/12 detail
show spanning-tree summary
debug spanning-tree events

📝 Practice Questions

Q1: What is the default STP priority on Cisco switches?
Answer: 32768. Lower priority = more likely to become root bridge.
Q2: How long does classic STP take to converge after a link failure?
Answer: Approximately 50 seconds (20 sec Blocking → 15 sec Listening → 15 sec Learning → Forwarding).
Q3: What command enables PortFast on an interface?
Answer: spanning-tree portfast
Q4: Why would you configure BPDUGuard on an access port?
Answer: To prevent unauthorized switches from being connected — if a BPDU is received, the port is shut down.
Q5: What is the main advantage of RSTP over classic STP?
Answer: Much faster convergence (< 6 seconds vs ~50 seconds).

🎓 Key Takeaways (STP Mastery)

  • ✅ STP prevents loops — essential for redundant network design
  • ✅ Root bridge election — lowest Bridge ID (priority + MAC) wins
  • ✅ Port states — Blocking → Listening → Learning → Forwarding (~50 sec)
  • ✅ PortFast — skip states for end devices (access ports only!)
  • ✅ BPDUGuard — security feature to block unauthorized switches
  • ✅ RSTP — modern replacement, ~6 sec convergence, backward compatible
  • ✅ MST — multiple VLANs per STP instance (reduces CPU load)
